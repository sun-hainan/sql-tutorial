## 1. 索引的概念和作用

### 🏠 生活类比

> **索引** = 书的目录：没有目录要一页页找，有目录直接翻到对应页

### 📖 概念讲解

| 项目 | 说明 |
|------|------|
| 索引 | 数据结构，帮助数据库快速定位数据 |
| 无索引查询 | O(n) 全表扫描，逐行检查 |
| 有索引查询 | O(log n) 索引树查找 |
| 代价 | 占用存储空间、增删改变慢（需维护索引结构） |

### 💻 SQL 代码示例

```sql
CREATE DATABASE index_demo;
USE index_demo;

CREATE TABLE t_user (
    id INT PRIMARY KEY,
    username VARCHAR(50),
    email VARCHAR(100),
    phone VARCHAR(20),
    age INT
);

-- 插入大量测试数据
INSERT INTO t_user (id, username, email, phone, age)
SELECT
    n,
    CONCAT('user_', n),
    CONCAT('user_', n, '@example.com'),
    CONCAT('138', LPAD(n, 8, '0')),
    FLOOR(18 + RAND() * 40)
FROM (
    SELECT a.n + b.n * 100 + c.n * 1000 AS n FROM
    (SELECT 0 AS n UNION SELECT 1 UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION
     SELECT 5 UNION SELECT 6 UNION SELECT 7 UNION SELECT 8 UNION SELECT 9) a,
    (SELECT 0 AS n UNION SELECT 1 UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION
     SELECT 5 UNION SELECT 6 UNION SELECT 7 UNION SELECT 8 UNION SELECT 9) b,
    (SELECT 0 AS n UNION SELECT 1 UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION
     SELECT 5 UNION SELECT 6 UNION SELECT 7 UNION SELECT 8 UNION SELECT 9) c
) t;

-- 无索引查询（type=ALL 表示全表扫描）
EXPLAIN SELECT * FROM t_user WHERE username = 'user_5000';

-- 创建索引
CREATE INDEX idx_username ON t_user(username);
CREATE INDEX idx_phone ON t_user(phone);

-- 有索引查询（type=ref 表示索引查找）
EXPLAIN SELECT * FROM t_user WHERE username = 'user_5000';
```

### ❌ 常见错误

- ❌ 所有列都建索引 → 写入性能严重下降，索引维护成本高
- ❌ 索引越多越好 → 占用空间，INSERT/UPDATE/DELETE 变慢

## 2. 索引的分类

### 📖 概念讲解

| 类型 | 说明 | 创建方式 |
|------|------|---------|
| 主键索引 | 唯一且非空，每表一个 | PRIMARY KEY |
| 唯一索引 | 值唯一，可为 NULL | UNIQUE INDEX |
| 普通索引 | 无唯一性限制 | INDEX / KEY |
| 全文索引 | 大文本搜索 | FULLTEXT INDEX |
| 复合索引 | 多列组合 | INDEX(col1, col2) |

### 💻 SQL 代码示例

```sql
USE index_demo;

CREATE TABLE t_employee (
    id INT PRIMARY KEY,                    -- 主键索引
    emp_code VARCHAR(20) UNIQUE,           -- 唯一索引
    emp_name VARCHAR(50),
    dept_id INT,
    email VARCHAR(100),
    resume TEXT,
    INDEX idx_name (emp_name),             -- 普通索引
    INDEX idx_dept (dept_id),             -- 普通索引
    FULLTEXT INDEX idx_resume (resume)     -- 全文索引
);

-- 复合索引
CREATE TABLE t_order (
    id INT PRIMARY KEY,
    order_date DATE,
    customer_id INT,
    status VARCHAR(20),
    INDEX idx_order_composite (order_date, customer_id, status)
);
INSERT INTO t_order VALUES
    (1, '2024-01-15', 101, 'paid'),
    (2, '2024-01-15', 102, 'pending'),
    (3, '2024-01-16', 101, 'shipped');

-- 复合索引生效情况
EXPLAIN SELECT * FROM t_order WHERE order_date = '2024-01-15';             -- ✅ 使用索引
EXPLAIN SELECT * FROM t_order WHERE order_date = '2024-01-15' AND customer_id = 101;  -- ✅ 使用索引
EXPLAIN SELECT * FROM t_order WHERE customer_id = 101;                        -- ❌ 不使用索引（跳过第一列）
```

## 3. 聚簇索引和非聚簇索引

### 🏠 生活类比

> **聚簇索引** = 按姓氏笔画排的字典：字的解释就存在对应位置（数据和索引在一起）
> **非聚簇索引** = 普通字典：按拼音排目录，但字的解释在另一区域（索引和数据分开）

### 📖 概念讲解

| 类型 | InnoDB | MyISAM |
|------|--------|--------|
| 主键索引 | 聚簇索引（数据存在叶子节点） | 非聚簇索引（索引叶子存数据地址） |
| 普通索引 | 非聚簇索引（叶子存主键值） | 非聚簇索引（叶子存数据地址） |

**回表**：通过普通索引找到主键值，再通过主键到聚簇索引查找完整数据

### 💻 SQL 代码示例

```sql
USE index_demo;

-- 主键是聚簇索引
CREATE TABLE t_goods (
    id INT PRIMARY KEY,           -- id 是聚簇索引，数据按 id 顺序存储
    name VARCHAR(50),
    price DECIMAL(10,2)
);
INSERT INTO t_goods VALUES (3, 'Mouse', 99), (1, 'Laptop', 5000), (2, 'Keyboard', 299);
-- 数据按主键 id 排序：1(Laptop), 2(Keyboard), 3(Mouse)

-- 普通索引是非聚簇索引
CREATE INDEX idx_name ON t_goods(name);

-- 覆盖索引：查询的列都在索引中，不需要回表
SELECT name FROM t_goods WHERE name = 'Mouse';
-- EXPLAIN 显示 Using index，表示覆盖索引
```

## 4. B+ 树原理

### 📖 概念讲解

**B+ 树**是 InnoDB 的索引结构：

| 特性 | 说明 |
|------|------|
| 多路搜索树 | 每个节点可有多个子节点，扇出大，树层数少 |
| 叶子节点存储数据 | 所有数据存在叶子节点，叶子节点用链表连接 |
| 适合范围查询 | 叶子节点链表支持范围遍历 |
| 磁盘 I/O 优化 | 每次 I/O 读取一个页（16KB），减少 I/O 次数 |

### 📊 ASCII B+ 树示意图

```
              [ 50 | 100 ]
            /     |     \
    [20|30]    [60|80]    [120|150]
      /  \        / \         / \
    数据   数据   数据  数据  数据  数据
    链表→→→→→→→→→→→→→→→→→→→→
```

## 5. 最左前缀原则

### 📖 概念讲解

**复合索引(col1, col2, col3)** 的使用规则：

| WHERE 条件 | 是否使用索引 |
|-----------|------------|
| col1 = 1 | ✅ |
| col1 = 1 AND col2 = 2 | ✅ |
| col1 = 1 AND col2 = 2 AND col3 = 3 | ✅ |
| col2 = 2 | ❌ |
| col3 = 3 | ❌ |

### 💻 SQL 代码示例

```sql
USE index_demo;

CREATE TABLE t_test_leftmost (
    id INT PRIMARY KEY,
    col1 INT,
    col2 INT,
    col3 INT,
    INDEX idx_composite (col1, col2, col3)
);
INSERT INTO t_test_leftmost VALUES
    (1, 1, 1, 1), (2, 1, 2, 2), (3, 2, 2, 3), (4, 2, 3, 4);

-- 索引生效
EXPLAIN SELECT * FROM t_test_leftmost WHERE col1 = 1;               -- ✅ 使用 col1
EXPLAIN SELECT * FROM t_test_leftmost WHERE col1 = 1 AND col2 = 2; -- ✅ 使用 col1+col2

-- 索引不生效
EXPLAIN SELECT * FROM t_test_leftmost WHERE col2 = 2;               -- ❌ 跳过 col1
EXPLAIN SELECT * FROM t_test_leftmost WHERE col3 = 3;               -- ❌ 跳过 col1, col2

-- 范围查询后列失效
EXPLAIN SELECT * FROM t_test_leftmost WHERE col1 = 1 AND col2 > 2 AND col3 = 3;
-- col3 无法使用索引，因为 col2 是范围查询
```

### ❌ 常见错误

- ❌ 创建复合索引后以为所有查询都能用 → 必须从最左列开始

## 6. 回表和覆盖索引

### 📖 概念讲解

| 概念 | 说明 |
|------|------|
| 回表 | 通过普通索引找到主键，再去聚簇索引查完整数据 |
| 覆盖索引 | 查询的列都在索引中，无需回表 |

### 💻 SQL 代码示例

```sql
USE index_demo;

CREATE TABLE t_sales_record (
    id INT PRIMARY KEY,
    sale_date DATE,
    product_name VARCHAR(50),
    amount DECIMAL(10,2)
);
CREATE INDEX idx_sale_date ON t_sales_record(sale_date);

-- 需要回表
EXPLAIN SELECT * FROM t_sales_record WHERE sale_date = '2024-01-15';

-- 覆盖索引：包含所有需要的列
CREATE INDEX idx_sale_date_cover ON t_sales_record(sale_date, product_name, amount);

EXPLAIN SELECT sale_date, product_name, amount FROM t_sales_record WHERE sale_date = '2024-01-15';
-- Using index，表示覆盖索引，无需回表
```

### ❌ 常见错误

- ❌ SELECT * 导致回表 → 应该只查需要的列

## 7. 索引失效场景

### 📖 概念讲解

| 场景 | 示例 |
|------|------|
| 列参与计算 | `WHERE age + 1 = 30` |
| 使用函数 | `WHERE YEAR(create_time) = 2024` |
| 类型转换 | `WHERE code = 123`（code 是 VARCHAR） |
| LIKE 以 % 开头 | `WHERE name LIKE '%xxx%'` |
| OR 连接非索引列 | `WHERE col1 = 'a' OR col2 = 'b'`（col2 无索引） |
| 不满足最左前缀 | `WHERE col2 = 2`（跳过第一列） |

### 💻 SQL 代码示例

```sql
USE index_demo;

CREATE TABLE t_idx_test (
    id INT PRIMARY KEY,
    code VARCHAR(20),
    amount INT,
    create_time DATETIME
);
CREATE INDEX idx_code ON t_idx_test(code);
CREATE INDEX idx_amount ON t_idx_test(amount);
CREATE INDEX idx_create_time ON t_idx_test(create_time);

-- 使用函数 → 不使用索引
EXPLAIN SELECT * FROM t_idx_test WHERE YEAR(create_time) = 2024;

-- LIKE 以 % 开头 → 不使用索引
EXPLAIN SELECT * FROM t_idx_test WHERE code LIKE '%123%';

-- LIKE 以 % 结尾 → 使用索引
EXPLAIN SELECT * FROM t_idx_test WHERE code LIKE '123%';
```

### ❌ 常见错误

- ❌ 在索引列上使用函数 → 索引失效

## 8. 查看和优化索引

### 💻 SQL 代码示例

```sql
USE index_demo;

-- 查看表的所有索引
SHOW INDEX FROM t_user;
SHOW INDEX FROM t_employee;

-- EXPLAIN 分析查询计划
EXPLAIN SELECT * FROM t_user WHERE username = 'user_5000';
EXPLAIN SELECT * FROM t_user WHERE phone LIKE '138%';
EXPLAIN SELECT * FROM t_user WHERE age > 30;
```

### ❌ 常见错误

- ❌ 以为 EXPLAIN 只是解释计划 → MySQL 8.0.18+ 的 EXPLAIN ANALYZE 会实际执行
