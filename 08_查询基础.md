- 掌握 SELECT 语句的八大子句执行顺序

- 掌握 ORDER BY 多字段排序和升序/降序

## 1. SELECT 执行顺序（核心！）

### 🏠 生活类比

> SQL 执行顺序就像做一道菜：
> 1. 去菜市场买菜（FROM）
> 2. 摘菜洗菜（WHERE）
> 3. 按菜分类摆放（GROUP BY）
> 4. 挑出好菜（HAVING）
> 5. 烹饪（SELECT）
> 6. 摆盘（ORDER BY）
> 7. 上菜（LIMIT）

### 📖 概念讲解

**SELECT 实际执行顺序**：

```
① FROM       → 确定数据来源，加载表
② ON         → 处理 JOIN 条件
③ JOIN       → 拼接表
④ WHERE      → 过滤数据（此时还不知道聚合结果）
⑤ GROUP BY   → 分组
⑥ HAVING     → 过滤分组结果
⑦ SELECT     → 选取列，计算表达式和聚合函数
⑧ DISTINCT   → 去重
⑨ ORDER BY   → 排序
⑩ LIMIT/TOP  → 分页/限制行数
```

**关键结论**：
- WHERE 在 GROUP BY 之前执行 → **WHERE 中不能用聚合函数**
- HAVING 在 GROUP BY 之后执行 → **HAVING 中可以用聚合函数**
- ORDER BY 在 SELECT 之后执行 → **可以用 SELECT 的别名排序**
- WHERE 先于 SELECT 执行 → **不能用 WHERE 过滤 SELECT 的别名**

### 💻 SQL 代码示例

```sql
CREATE DATABASE select_demo;
USE select_demo;

CREATE TABLE t_score (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_name VARCHAR(50),
    subject VARCHAR(20),
    score INT,
    class_id INT
);
INSERT INTO t_score (student_name, subject, score, class_id) VALUES
    ('Alice', 'Math', 85, 1),
    ('Bob', 'Math', 92, 1),
    ('Carol', 'Math', 78, 2),
    ('Alice', 'English', 90, 1),
    ('Bob', 'English', 88, 1),
    ('Carol', 'English', 95, 2),
    ('David', 'Math', 70, 2),
    ('Eve', 'English', 82, 1);

-- 完整查询演示
SELECT subject, AVG(score) AS avg_score
FROM t_score
WHERE class_id = 1          -- ① WHERE 过滤
GROUP BY subject             -- ② GROUP BY 分组
HAVING AVG(score) > 85      -- ③ HAVING 过滤分组
ORDER BY avg_score DESC      -- ④ 按别名排序
LIMIT 1;                     -- ⑤ 取第一条

-- 常见错误：WHERE 中使用聚合函数
-- SELECT * FROM t_score WHERE AVG(score) > 85;  -- Error!
-- 正确做法：用 HAVING
SELECT subject, AVG(score) AS avg_score
FROM t_score
GROUP BY subject
HAVING AVG(score) > 85;
```

### 📊 ASCII 执行流程图

```
FROM t_score
    ↓
WHERE class_id = 1  ────────── 过滤：class_id=1
    ↓
GROUP BY subject   ────────── 分组：Math(2人), English(2人)
    ↓
HAVING AVG(score)>85 ─────── 过滤：Math平均88.5✅, English平均86.7✅
    ↓
SELECT subject, AVG(score) ── 选取
    ↓
ORDER BY avg_score DESC ──── 排序
    ↓
LIMIT 1             ──────── 取1条
```

### ❌ 常见错误

- ❌ 在 WHERE 中使用 SELECT 的别名 → 别名还没生成，应该在 HAVING 或 ORDER BY 中使用
- ❌ 在 WHERE 中使用聚合函数 → 应该用 HAVING

## 2. WHERE 子句详解

### 🏠 生活类比

> **WHERE** = 筛子：把不符合条件的豆子筛掉，只留下符合的

### 📖 概念讲解

| 操作符 | 说明 |
|--------|------|
| `=` / `<>` | 等于/不等于 |
| `>` / `<` / `>=` / `<=` | 比较 |
| AND / OR / NOT | 逻辑组合 |
| BETWEEN ... AND ... | 闭区间（包含两端） |
| IN (...) | 在列表中 |
| LIKE '%xxx%' | 模糊匹配（%任意字符） |
| IS NULL / IS NOT NULL | 空值判断 |

### 💻 SQL 代码示例

```sql
USE select_demo;

-- 基础比较
SELECT * FROM t_score WHERE score >= 80;
SELECT * FROM t_score WHERE student_name = 'Alice';

-- AND / OR 组合
SELECT * FROM t_score WHERE subject = 'Math' AND score > 80;
SELECT * FROM t_score WHERE subject = 'Math' OR subject = 'English';

-- BETWEEN（闭区间：a ≤ x ≤ b）
SELECT * FROM t_score WHERE score BETWEEN 75 AND 90;

-- IN（代替多个 OR）
SELECT * FROM t_score WHERE student_name IN ('Alice', 'Bob', 'Carol');

-- LIKE 模糊匹配
SELECT * FROM t_score WHERE student_name LIKE 'A%';     -- 以A开头的名字
SELECT * FROM t_score WHERE student_name LIKE '%e%';    -- 包含e的名字
SELECT * FROM t_score WHERE student_name LIKE 'C_rl';  -- C开头+任意字符+r+任意字符+l（Carol）

-- IS NULL / IS NOT NULL
SELECT * FROM t_score WHERE score IS NULL;
SELECT * FROM t_score WHERE score IS NOT NULL;

-- NOT 取反
SELECT * FROM t_score WHERE score NOT BETWEEN 70 AND 90;
SELECT * FROM t_score WHERE student_name NOT IN ('Alice', 'Bob');

-- 组合复杂条件
SELECT * FROM t_score
WHERE (subject = 'Math' AND score > 80) OR (subject = 'English' AND score > 90);
```

### ❌ 常见错误

- ❌ 忘记给字符串加引号 → `'Alice'` 而非 `Alice`
- ❌ AND/OR 优先级混乱 → 应该用括号明确优先级

## 3. ORDER BY 多字段排序

### 🏠 生活类比

> **多字段排序** = 考试成绩排名：先按总分排，总分相同按语文成绩排

### 📖 概念讲解

- `ASC` = 升序（默认，从小到大）
- `DESC` = 降序（从大到小）
- 多字段排序：先按第一字段排，相等时按第二字段排
- NULL 值在 ASC 中排在最前，DESC 中排在最后

### 💻 SQL 代码示例

```sql
USE select_demo;

-- 单字段升序（默认）
SELECT * FROM t_score ORDER BY score ASC;
SELECT * FROM t_score ORDER BY score;  -- ASC 可省略

-- 单字段降序
SELECT * FROM t_score ORDER BY score DESC;

-- 多字段排序：先按 subject，再按 score
SELECT * FROM t_score ORDER BY subject ASC, score DESC;
-- 先按 subject 排序，所有 Math 排在一起，English 排在一起
-- 同科目内按 score 降序排

-- 表达式排序
SELECT student_name, score, score * 1.1 AS curved_score
FROM t_score
ORDER BY curved_score DESC;

-- 按别名排序（因为 ORDER BY 在 SELECT 之后执行）
SELECT subject, AVG(score) AS avg_score
FROM t_score
GROUP BY subject
ORDER BY avg_score DESC;

-- NULL 值排序位置
CREATE TABLE t_null_sort (
    id INT,
    val INT
);
INSERT INTO t_null_sort VALUES (1, 10), (2, NULL), (3, 20), (4, NULL);
SELECT * FROM t_null_sort ORDER BY val ASC;              -- NULL 在最前
SELECT * FROM t_null_sort ORDER BY val DESC;             -- NULL 在最后

-- MySQL 处理 NULL：使用 IFNULL
SELECT * FROM t_null_sort ORDER BY IFNULL(val, 999999) ASC;
```

### ❌ 常见错误

- ❌ ORDER BY 的别名在 SELECT 中没有 → 报错

## 4. DISTINCT 和 LIMIT

### 📖 概念讲解

| 关键字 | 作用 | 示例 |
|--------|------|------|
| DISTINCT | 去除重复行 | SELECT DISTINCT subject FROM t_score |
| LIMIT n | 返回前 n 条 | SELECT * FROM t_score LIMIT 5 |
| LIMIT m, n | 跳过 m 条，返回 n 条 | SELECT * FROM t_score LIMIT 3, 3（第二页） |

### 💻 SQL 代码示例

```sql
USE select_demo;

-- DISTINCT 基础
SELECT DISTINCT subject FROM t_score;     -- 去重后科目
SELECT DISTINCT student_name FROM t_score;  -- 6个不同学生

-- DISTINCT 作用于多列组合（所有列的组合去重）
SELECT DISTINCT class_id, subject FROM t_score;

-- DISTINCT + 聚合函数
SELECT COUNT(DISTINCT student_name) AS student_count FROM t_score;
SELECT COUNT(DISTINCT class_id) AS class_count FROM t_score;
SELECT COUNT(DISTINCT class_id, subject) AS distinct_combinations FROM t_score;

-- LIMIT 基础
SELECT * FROM t_score LIMIT 5;

-- LIMIT + OFFSET（分页：第2页，每页3条）
SELECT * FROM t_score ORDER BY id LIMIT 3 OFFSET 3;

-- 简写形式：LIMIT offset, count
SELECT * FROM t_score ORDER BY id LIMIT 6, 3;

-- DISTINCT + LIMIT
SELECT DISTINCT student_name FROM t_score LIMIT 3;  -- 先去重，再取前3
```

### ❌ 常见错误

- ❌ SELECT * DISTINCT ... → DISTINCT 应该在 SELECT 后面

## 5. 别名（AS）

### 🏠 生活类比

> **别名** = 给人起外号：本来叫"张伟"，外号叫"小张"，指的是同一个人

### 📖 概念讲解

- 列别名用于 ORDER BY、HAVING 的引用（SELECT 之后执行）
- 表别名在 FROM 子句中使用，作用域是整条 SQL
- 如果别名含空格或关键字，需用引号

### 💻 SQL 代码示例

```sql
USE select_demo;

-- 列别名
SELECT
    student_name AS name,
    score AS exam_score,
    score * 1.1 AS curved_score
FROM t_score;

-- 表别名（JOIN 时必须用）
SELECT
    s.student_name,
    s.score
FROM t_score AS s
WHERE s.score > 80;

-- 别名用于 ORDER BY
SELECT
    subject,
    AVG(score) AS avg_score,
    MAX(score) AS max_score
FROM t_score
GROUP BY subject
ORDER BY avg_score DESC;

-- 别名用于 HAVING
SELECT
    student_name,
    AVG(score) AS avg_score
FROM t_score
GROUP BY student_name
HAVING avg_score > 85;

-- 多表 JOIN 时用别名防列名冲突
CREATE TABLE t_class (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
INSERT INTO t_class VALUES (1, 'Class A'), (2, 'Class B');

SELECT s.student_name, c.name AS class_name
FROM t_score s
LEFT JOIN t_class c ON s.class_id = c.id;
```

### ❌ 常见错误

- ❌ WHERE 中使用列别名 → WHERE 先于 SELECT 执行，别名还不存在

## 6. UNION 和 UNION ALL

### 🏠 生活类比

> **UNION** = 把两份名单合并，去掉重复的名字
> **UNION ALL** = 把两份名单直接拼接，保留所有名字

### 📖 概念讲解

| 操作符 | 去重 | 性能 |
|--------|------|------|
| UNION | ✅ 自动去重 | 较慢（多一步去重） |
| UNION ALL | ❌ 不去重 | 较快（直接追加） |

### 💻 SQL 代码示例

```sql
USE select_demo;

-- UNION 去重
SELECT student_name FROM t_score WHERE class_id = 1
UNION
SELECT student_name FROM t_score WHERE subject = 'English';

-- UNION ALL 保留重复
SELECT student_name FROM t_score WHERE class_id = 1
UNION ALL
SELECT student_name FROM t_score WHERE subject = 'English';

-- UNION 配合 ORDER BY（整个并集排序）
SELECT student_name, score FROM t_score WHERE class_id = 1
UNION ALL
SELECT student_name, score FROM t_score WHERE class_id = 2
ORDER BY score DESC;

-- 合并不同类型的数据
CREATE TABLE t_teacher (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
INSERT INTO t_teacher VALUES (1, 'Teacher Wang'), (2, 'Teacher Li');

SELECT student_name AS name FROM t_score
UNION
SELECT name FROM t_teacher
ORDER BY name;
```

### ❌ 常见错误

- ❌ UNION ALL 当 UNION 用 → 不知道有重复数据时没问题，但浪费了去重操作

## 7. 子查询

### 🏠 生活类比

> **子查询** = 套娃：先回答里面的小问题，再回答外面的问题

### 📖 概念讲解

| 类型 | 返回值 | 示例 |
|------|--------|------|
| 标量子查询 | 单个值 | `WHERE score > (SELECT AVG(score) FROM t)` |
| 列子查询 | 一列值 | `WHERE name IN (SELECT name FROM t2)` |
| 行子查询 | 一行 | `WHERE (col1, col2) = (SELECT col1, col2 FROM t2)` |
| 表子查询 | 临时表 | `FROM (SELECT ...) AS sub` |

### 💻 SQL 代码示例

```sql
USE select_demo;

-- 标量子查询：返回单个值
SELECT * FROM t_score
WHERE score > (SELECT AVG(score) FROM t_score);  -- 高于平均分的成绩

-- 列子查询：返回一列值
SELECT * FROM t_score
WHERE student_name IN (
    SELECT DISTINCT student_name FROM t_score WHERE score > 85
);

-- 行子查询：返回一行
CREATE TABLE t_best_student (
    student_name VARCHAR(50),
    subject VARCHAR(20),
    score INT
);
INSERT INTO t_best_student VALUES ('Alice', 'English', 90);
SELECT * FROM t_best_student
WHERE (student_name, score) = (
    SELECT student_name, MAX(score) FROM t_score
);

-- FROM 子查询（派生表）
SELECT avg_score FROM (
    SELECT student_name, AVG(score) AS avg_score
    FROM t_score
    GROUP BY student_name
) AS student_avg
WHERE avg_score > 85;

-- SELECT 中的子查询
SELECT
    student_name,
    score,
    (SELECT AVG(score) FROM t_score WHERE student_name = s.student_name) AS personal_avg,
    (SELECT MAX(score) FROM t_score) AS global_max
FROM t_score s;

-- EXISTS 子查询
SELECT * FROM t_score s
WHERE EXISTS (
    SELECT 1 FROM t_score WHERE student_name = s.student_name AND score > 90
);
```

### ❌ 常见错误

- ❌ 子查询返回多行但用 `=` 比较 → 应该用 IN

## 本章小结

```
┌─────────────────────────────────────────────────────────┐
│                 SELECT 执行顺序（重要！）                 │
├─────────────────────────────────────────────────────────┤
│ FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
│                                                          │
│ WHERE：过滤行（不能用聚合函数、不能用别名）              │
│ GROUP BY：分组                                           │
│ HAVING：过滤分组（可以用聚合函数）                      │
│ SELECT：选取列（可以计算聚合、可以用别名）              │
│ ORDER BY：排序（可以用别名）                            │
│ LIMIT：分页                                              │
│                                                          │
│ DISTINCT：去重（基于所有 SELECT 列）                    │
│ UNION vs UNION ALL：去重 vs 不去重                      │
│                                                          │
│ 子查询类型：标量/列/行/表子查询                          │
│ IN vs EXISTS：列表匹配 vs 存在性检查                     │
└─────────────────────────────────────────────────────────┘
```
