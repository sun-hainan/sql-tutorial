# 模块十二：视图

## 学习目标

- 理解视图的概念和作用
- 掌握创建、查看、删除视图的方法
- 理解可更新视图和不可更新视图的区别
- 掌握 WITH CHECK OPTION 的作用
- 理解视图在安全性和简化查询中的应用

---

## 1. 视图的概念和创建

### 🏠 生活类比

> **视图** = 会议室的预约系统：会议室是真实存在的桌子（基表），预约系统是视图（虚拟的，只显示谁在用）

### 📖 概念讲解

视图（VIEW）是预先定义好的 SQL 查询，不存储实际数据。每次访问视图时，视图的 SELECT 查询实时执行。

**视图的优点**：
- 简化复杂查询
- 隐藏数据复杂性
- 提高安全性（只暴露必要列）
- 代码复用

### 💻 SQL 代码示例

```sql
CREATE DATABASE view_demo;
USE view_demo;

CREATE TABLE t_dept (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50)
);
CREATE TABLE t_emp (
    emp_id INT PRIMARY KEY,
    emp_name VARCHAR(50),
    salary DECIMAL(10,2),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES t_dept(dept_id)
);
INSERT INTO t_dept VALUES (1, 'IT'), (2, 'HR'), (3, 'Finance');
INSERT INTO t_emp VALUES
    (1, 'Alice', 8000, 1), (2, 'Bob', 7000, 1),
    (3, 'Carol', 6000, 2), (4, 'David', 7500, 2),
    (5, 'Eve', 9000, 1), (6, 'Frank', 5500, 3);

-- 创建视图：IT部门员工
CREATE VIEW v_it_employees AS
SELECT emp_id, emp_name, salary
FROM t_emp
WHERE dept_id = 1;

-- 使用视图
SELECT * FROM v_it_employees;
SELECT emp_name, salary * 1.1 AS increased_salary FROM v_it_employees;
```

### ❌ 常见错误

- ❌ 以为视图是真实表 → 视图是虚拟的，每次查询实时计算
- ❌ 视图索引查询慢 → 视图不存储数据，无法建索引

## 2. 可更新视图

### 🏠 生活类比

> **可更新视图** = 可以直接修改会议室预约系统里的信息（会同步到真实系统）

### 📖 概念讲解

| 视图类型 | 说明 | 可否 UPDATE/DELETE |
|---------|------|-------------------|
| 简单视图 | 单表、无聚合、无 GROUP BY | ✅ 可更新 |
| 复杂视图 | 含聚合/GROUP BY/JOIN | ❌ 不可更新 |

**可更新视图条件**：
1. 不含聚合函数、DISTINCT、GROUP BY、HAVING、UNION
2. 不含子查询（MySQL 5.7+ 部分支持）
3. 不含 JOIN（MySQL 5.7+ 部分支持）
4. 必须包含基表的所有 NOT NULL 列

### 💻 SQL 代码示例

```sql
USE view_demo;

-- 简单视图（可更新）
CREATE VIEW v_high_salary AS
SELECT emp_id, emp_name, salary
FROM t_emp
WHERE salary > 6000;

-- 通过视图更新数据
UPDATE v_high_salary SET salary = 8500 WHERE emp_id = 1;
SELECT * FROM t_emp WHERE emp_id = 1;  -- 视图结果同步到基表

-- 通过视图删除数据
DELETE FROM v_high_salary WHERE emp_id = 6;

-- 复杂视图（不可更新）
CREATE VIEW v_dept_summary AS
SELECT dept_id, COUNT(*) AS emp_count, AVG(salary) AS avg_salary
FROM t_emp
GROUP BY dept_id;
-- INSERT INTO v_dept_summary VALUES (4, 10, 7000);  -- Error
```

### ❌ 常见错误

- ❌ 对复杂视图执行 INSERT/UPDATE → 会报错

## 3. WITH CHECK OPTION

### 🏠 生活类比

> **WITH CHECK OPTION** = 会议室预约系统禁止预约超出权限的时间段

### 📖 概念讲解

默认情况下，通过视图插入不符合视图 WHERE 条件的数据，实际会写入基表，只是查询视图时看不到。

**WITH CHECK OPTION** 强制要求 INSERT/UPDATE 的数据必须满足视图的 WHERE 条件。

### 💻 SQL 代码示例

```sql
USE view_demo;

-- 不带 CHECK OPTION 的视图
CREATE VIEW v_it_staff AS
SELECT emp_id, emp_name, salary
FROM t_emp
WHERE dept_id = 1;

-- 插入IT部门以外的数据（基表有，但视图看不到）
INSERT INTO v_it_staff VALUES (10, 'Test', 5000);
-- SELECT * FROM v_it_staff;  -- 看不到Test

-- 带 WITH CHECK OPTION 的视图
CREATE VIEW v_it_staff_checked AS
SELECT emp_id, emp_name, salary
FROM t_emp
WHERE dept_id = 1
WITH CHECK OPTION;

-- 尝试插入不符合 dept_id=1 的数据
-- INSERT INTO v_it_staff_checked VALUES (11, 'Test2', 5000);  -- Error

-- 插入符合条件的数据，可以成功
INSERT INTO v_it_staff_checked VALUES (11, 'IT_New', 7000);
SELECT * FROM v_it_staff_checked;
```

### ❌ 常见错误

- ❌ 以为插入视图的数据都在视图里 → 默认情况下不符合 WHERE 的数据也能插入

## 4. 视图在安全性和简化查询中的应用

### 📖 概念讲解

| 用途 | 说明 |
|------|------|
| 隐藏敏感列 | 只暴露必要列，隐藏工资等 |
| 简化复杂查询 | 把 JOIN 封装成视图 |
| 权限控制 | 用户只能通过视图操作数据 |

### 💻 SQL 代码示例

```sql
USE view_demo;

-- 安全性示例：隐藏工资列
CREATE VIEW v_emp_public AS
SELECT emp_id, emp_name, dept_id
FROM t_emp;
-- 用户只能看到 emp_id, emp_name, dept_id，看不到 salary

-- 简化查询示例：封装常用的部门统计
CREATE VIEW v_dept_stats AS
SELECT
    d.dept_name,
    COUNT(e.emp_id) AS emp_count,
    SUM(e.salary) AS total_salary,
    AVG(e.salary) AS avg_salary
FROM t_dept d
LEFT JOIN t_emp e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name;

-- 之前：复杂 JOIN + 聚合
-- 现在：直接查视图
SELECT * FROM v_dept_stats;
SELECT * FROM v_dept_stats WHERE dept_name = 'IT';
```

### ❌ 常见错误

- ❌ 以为视图可以完全替代权限控制 → 视图只是辅助，真正的权限控制用 GRANT

## 5. 视图的优缺点

### 📖 概念讲解

| 优点 | 缺点 |
|------|------|
| 简化复杂查询 | 每次查询都要重新计算 |
| 增强安全性 | 无法建索引 |
| 提供逻辑独立性 | 不支持所有 SQL 操作 |
| 代码复用 | 依赖基表 |

### 💻 SQL 代码示例

```sql
USE view_demo;

-- 复杂视图（性能开销大）
CREATE VIEW v_complex AS
SELECT
    e.emp_id,
    e.emp_name,
    e.salary,
    d.dept_name,
    (SELECT AVG(salary) FROM t_emp WHERE dept_id = e.dept_id) AS dept_avg_salary
FROM t_emp e
INNER JOIN t_dept d ON e.dept_id = d.dept_id;

-- EXPLAIN 分析：视图查询会执行子查询
EXPLAIN SELECT * FROM v_complex WHERE dept_name = 'IT';

-- 基表变更影响视图
ALTER TABLE t_emp ADD COLUMN email VARCHAR(100);
SHOW CREATE VIEW v_complex;  -- 视图定义不变，但依赖原列
```

### ❌ 常见错误

- ❌ 滥用视图 → 复杂视图频繁查询时性能差

## 6. 查看、修改和删除视图

### 💻 SQL 代码示例

```sql
USE view_demo;

-- 查看视图定义
SHOW CREATE VIEW v_it_employees;
SHOW CREATE TABLE v_it_employees;  -- 也可以用 SHOW CREATE TABLE

-- 查看当前数据库所有视图
SHOW TABLE STATUS WHERE Comment = 'VIEW';

-- 修改视图（方式1：CREATE OR REPLACE）
CREATE OR REPLACE VIEW v_it_employees AS
SELECT emp_id, emp_name, salary, dept_id
FROM t_emp
WHERE dept_id = 1
WITH CHECK OPTION;

-- 修改视图（方式2：ALTER）
ALTER VIEW v_it_employees AS
SELECT emp_id, emp_name, salary
FROM t_emp
WHERE dept_id = 1;

-- 删除视图
DROP VIEW IF EXISTS v_it_employees;
DROP VIEW IF EXISTS v_dept_summary;

-- 查看视图
SHOW FULL TABLES WHERE Table_type = 'VIEW';
```

### ❌ 常见错误

- ❌ 误删视图 → DROP VIEW 只是删除视图定义，不影响基表数据

## 本章小结

```
┌─────────────────────────────────────────────────────────┐
│                    视图核心                               │
├─────────────────────────────────────────────────────────┤
│ 视图 = 预定义的 SQL 查询，不存储实际数据                 │
│                                                          │
│ 可更新视图条件：                                         │
│   单表、无聚合、无 GROUP BY、无 DISTINCT              │

│   防止插入/更新不符合 WHERE 条件的数据                  │

│ 用途：                                                  │
│   简化查询 / 隐藏敏感列 / 权限控制                      │

│ 优缺点：                                                │
│   ✅ 简化查询、✅ 提高安全性、✅ 代码复用               │
│   ❌ 性能开销、❌ 无索引、❌ 复杂视图不可更新         │
└─────────────────────────────────────────────────────────┘
```
