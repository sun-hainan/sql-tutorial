# 模块七：增删改数据

## 学习目标

- 掌握 INSERT 的五种语法（单行/多行/INSERT SELECT/SET/REPLACE）
- 掌握 UPDATE 的用法及注意事项
- 掌握 DELETE 的用法及与 TRUNCATE 的区别
- 理解批量插入的原理和性能优化
- 掌握逻辑删除的设计方案
- 掌握 ON DUPLICATE KEY UPDATE（Upsert）用法

## 1. INSERT 的多种语法

### 🏠 生活类比

> **INSERT** = 往书架上放新书，可以一本一本放，也可以批量用箱子运

### 📖 概念讲解

| 语法 | 场景 |
|------|------|
| `INSERT ... VALUES (...)` | 单行插入 |
| `INSERT ... VALUES (...), (...), (...)` | 多行插入（高效） |
| `INSERT ... SELECT ...` | 从查询结果插入 |
| `INSERT ... SET col=val...` | 列名=值的形式 |
| `REPLACE INTO` | 唯一键冲突则先删后插 |
| `INSERT IGNORE` | 唯一键冲突则忽略 |

### 💻 SQL 代码示例

```sql
CREATE DATABASE dml_demo;
USE dml_demo;

CREATE TABLE t_student (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT DEFAULT 18,
    email VARCHAR(100) UNIQUE,
    status VARCHAR(20) DEFAULT 'active',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 1. 标准单行插入（指定所有列）
INSERT INTO t_student (name, age, email, status)
VALUES ('Alice', 20, 'alice@example.com', 'active');

-- 2. 省略列（使用默认值）
INSERT INTO t_student (name, email)
VALUES ('Bob', 'bob@example.com');
-- age=DEFAULT=18, status=DEFAULT='active'

-- 3. 多行插入（一次插入多条，高效）
INSERT INTO t_student (name, age, email) VALUES
    ('Carol', 22, 'carol@example.com'),
    ('David', 19, 'david@example.com'),
    ('Eve', 25, 'eve@example.com');

-- 4. 插入查询结果（INSERT SELECT）
CREATE TABLE t_student_backup LIKE t_student;
INSERT INTO t_student_backup
SELECT * FROM t_student WHERE status = 'active';

-- 5. SET 语法（列名=值的形式，更清晰）
INSERT INTO t_student SET
    name='Frank',
    email='frank@example.com',
    age=30;

-- 6. REPLACE INTO（唯一键冲突则先删后插）
CREATE TABLE t_session (
    session_id VARCHAR(50) PRIMARY KEY,
    user_id INT,
    last_active DATETIME
);
REPLACE INTO t_session VALUES ('ABC123', 1, NOW());
REPLACE INTO t_session VALUES ('ABC123', 2, NOW());  -- 先删后插
SELECT * FROM t_session;  -- 只有1条记录，user_id=2

-- 7. INSERT IGNORE（唯一键冲突则忽略）
INSERT IGNORE INTO t_student (name, email)
VALUES ('Alice2', 'alice@example.com');  -- email 重复，忽略

SELECT * FROM t_student;
```

### ❌ 常见错误

- ❌ INSERT 漏写列名 → 如果 DEFAULT 存在则用 DEFAULT，否则报错
- ❌ REPLACE 误用 → 会删除旧记录，导致自增 ID 变化

## 2. UPDATE 语句

### 🏠 生活类比

> **UPDATE** = 修改文件内容（可以改一处，也可以改多处，但**必须指定改哪**）

### 📖 概念讲解

- UPDATE 受事务控制，可以用 ROLLBACK 回滚
- **不带 WHERE = 更新整表，非常危险！**
- 可以配合 ORDER BY 和 LIMIT 实现部分更新

### 💻 SQL 代码示例

```sql
USE dml_demo;

CREATE TABLE t_product (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50),
    price DECIMAL(10,2),
    stock INT,
    status VARCHAR(20)
);
INSERT INTO t_product (name, price, stock, status) VALUES
    ('Laptop', 5000.00, 100, 'active'),
    ('Mouse', 99.00, 500, 'active'),
    ('Keyboard', 299.00, 200, 'discontinued'),
    ('Monitor', 1299.00, 50, 'active');

-- 基础更新
UPDATE t_product SET price = 4799.00 WHERE id = 1;

-- 批量更新（使用表达式）
UPDATE t_product
SET price = price * 0.9
WHERE status = 'active' AND price > 500;

-- 同时更新多个列
UPDATE t_product
SET price = 1099.00, stock = stock - 10
WHERE id = 4;

-- 使用 ORDER BY + LIMIT 限制更新行数
UPDATE t_product
SET status = 'pending_review'
WHERE status = 'active'
ORDER BY price ASC
LIMIT 2;

-- 子查询更新
UPDATE t_product
SET price = (SELECT MAX(price) * 0.8 FROM t_product)
WHERE id = 3;

-- 多表更新
CREATE TABLE t_sales (
    id INT PRIMARY KEY AUTO_INCREMENT,
    product_id INT,
    quantity INT
);
INSERT INTO t_sales VALUES (1, 1, 2), (2, 2, 5), (3, 3, 1);

UPDATE t_product p
INNER JOIN t_sales s ON p.id = s.product_id
SET p.stock = p.stock - s.quantity
WHERE s.id = 2;

SELECT * FROM t_product;

-- 测试回滚
START TRANSACTION;
UPDATE t_product SET price = price * 1.1;
ROLLBACK;
SELECT price FROM t_product WHERE id = 1;  -- 价格未变
```

### ❌ 常见错误

- ❌ UPDATE 不带 WHERE → 更新整表数据，非常危险！
- ❌ 在 UPDATE SET 中引用被更新的同一行 → 可能产生微妙副作用

## 3. DELETE 语句

### 🏠 生活类比

> **DELETE** = 撕掉文件的一页（可以指定撕哪页，不撕就不丢）

### 💻 SQL 代码示例

```sql
USE dml_demo;

CREATE TABLE t_log (
    id INT PRIMARY KEY AUTO_INCREMENT,
    action VARCHAR(50),
    log_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
INSERT INTO t_log (action) VALUES
    ('login'), ('page_view'), ('click'), ('purchase'), ('logout');

-- 带 WHERE 条件删除
DELETE FROM t_log WHERE action = 'page_view';

-- 删除满足条件的多行
DELETE FROM t_log WHERE action IN ('login', 'logout');

-- 按时间排序后删除最早的几条
DELETE FROM t_log ORDER BY log_time ASC LIMIT 1;

-- 全表删除（危险！但可以回滚）
DELETE FROM t_log;

-- DELETE vs TRUNCATE 对比
CREATE TABLE t_compare (
    id INT PRIMARY KEY AUTO_INCREMENT,
    val VARCHAR(50)
);
INSERT INTO t_compare VALUES (1, 'A'), (2, 'B'), (3, 'C');

-- DELETE：可回滚，保留 AUTO_INCREMENT
START TRANSACTION;
DELETE FROM t_compare;
ROLLBACK;
INSERT INTO t_compare VALUES (4, 'D');
SELECT * FROM t_compare;  -- id=4（AUTO_INCREMENT 未重置）

-- TRUNCATE：不可回滚，重置 AUTO_INCREMENT
TRUNCATE TABLE t_compare;
INSERT INTO t_compare VALUES (5, 'E');
SELECT * FROM t_compare;  -- id=1（AUTO_INCREMENT 重置）
```

### ❌ 常见错误

- ❌ DELETE 全表后以为能回滚 → 如果不在事务中则无法回滚

## 4. 批量插入

### 🏠 生活类比

> 逐条 INSERT = 一次买一件快递（10次运10次）
> 批量 INSERT = 一次买10件一起发（1次运10件，效率高10倍）

### 📖 概念讲解

**批量插入效率对比**：逐条 INSERT 1000次 vs 批量 INSERT 1次，后者快 10-50 倍。

百万级数据推荐：LOAD DATA INFILE

### 💻 SQL 代码示例

```sql
USE dml_demo;

CREATE TABLE t_order_batch (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_name VARCHAR(50),
    product_name VARCHAR(50),
    amount DECIMAL(10,2),
    order_date DATE
);

-- 低效方式：循环逐条插入（演示用）
DELIMITER //
CREATE PROCEDURE p_insert_batch_slow()
BEGIN
    DECLARE i INT DEFAULT 1;
    WHILE i <= 100 DO
        INSERT INTO t_order_batch (customer_name, product_name, amount, order_date)
        VALUES (CONCAT('Customer', i), CONCAT('Product', i), i * 10.00, CURDATE());
        SET i = i + 1;
    END WHILE;
END//
DELIMITER ;

CALL p_insert_batch_slow();
SELECT COUNT(*) FROM t_order_batch;

-- 高效方式：单条 SQL 批量插入（100条）
TRUNCATE TABLE t_order_batch;
INSERT INTO t_order_batch (customer_name, product_name, amount, order_date)
SELECT
    CONCAT('Customer', n),
    CONCAT('Product', n),
    n * 10.00,
    CURDATE()
FROM (
    SELECT 1 AS n UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION SELECT 5 UNION
    SELECT 6 UNION SELECT 7 UNION SELECT 8 UNION SELECT 9 UNION SELECT 10 UNION
    SELECT 11 UNION SELECT 12 UNION SELECT 13 UNION SELECT 14 UNION SELECT 15 UNION
    SELECT 16 UNION SELECT 17 UNION SELECT 18 UNION SELECT 19 UNION SELECT 20
) t;

SELECT COUNT(*) FROM t_order_batch;  -- 20条

-- 更大批量：1000条（10x10）
TRUNCATE TABLE t_order_batch;
INSERT INTO t_order_batch (customer_name, product_name, amount, order_date)
SELECT
    CONCAT('Customer', n),
    CONCAT('Product', n),
    n * 10.00,
    CURDATE()
FROM (
    SELECT a.n + b.n * 10 AS n FROM
    (SELECT 1 AS n UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION SELECT 5 UNION
     SELECT 6 UNION SELECT 7 UNION SELECT 8 UNION SELECT 9 UNION SELECT 10) a,
    (SELECT 1 AS n UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION SELECT 5 UNION
     SELECT 6 UNION SELECT 7 UNION SELECT 8 UNION SELECT 9 UNION SELECT 10) b
) t;

SELECT COUNT(*) FROM t_order_batch;  -- 100条
```

### ❌ 常见错误

- ❌ 循环中逐条 INSERT → 性能极差，应该用批量 INSERT

## 5. 逻辑删除

### 🏠 生活类比

> **逻辑删除** = 在文件上画删除线（还在，只是标记为"已删除"）
> **物理删除** = 把文件扔进垃圾桶（真的没了）

### 📖 概念讲解

| 类型 | 实现方式 | 数据恢复 | 查询性能 |
|------|---------|---------|---------|
| 逻辑删除 | UPDATE is_deleted=1 | ✅ 可恢复 | ❌ 需加条件 |
| 物理删除 | DELETE | ❌ 不可恢复 | ✅ 简单 |

### 💻 SQL 代码示例

```sql
USE dml_demo;

CREATE TABLE t_user_logic_del (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    is_deleted TINYINT DEFAULT 0,        -- 0=未删除，1=已删除
    delete_time DATETIME DEFAULT NULL
);

INSERT INTO t_user_logic_del (username, email) VALUES
    ('alice', 'alice@example.com'),
    ('bob', 'bob@example.com'),
    ('carol', 'carol@example.com');

-- 逻辑删除：UPDATE 而非 DELETE
UPDATE t_user_logic_del
SET is_deleted = 1, delete_time = NOW()
WHERE id = 2;

-- 查询有效数据（必须加 is_deleted 条件）
SELECT * FROM t_user_logic_del WHERE is_deleted = 0;

-- 恢复数据
UPDATE t_user_logic_del
SET is_deleted = 0, delete_time = NULL
WHERE id = 2;

SELECT * FROM t_user_logic_del WHERE is_deleted = 0;  -- Bob 恢复

-- 唯一约束与逻辑删除的冲突（解决：唯一约束不包含 is_deleted 时可以重新插入）
CREATE TABLE t_code_logic_del (
    id INT PRIMARY KEY AUTO_INCREMENT,
    code VARCHAR(20) NOT NULL,
    name VARCHAR(50),
    is_deleted TINYINT DEFAULT 0
);
INSERT INTO t_code_logic_del (code, name, is_deleted) VALUES ('ABC', 'First', 0);
-- 软删除
UPDATE t_code_logic_del SET is_deleted = 1 WHERE id = 1;
-- 重新插入相同 code（因为之前的 is_deleted=1）
INSERT INTO t_code_logic_del (code, name, is_deleted) VALUES ('ABC', 'New First', 0);

SELECT * FROM t_code_logic_del;
```

### ❌ 常见错误

- ❌ 逻辑删除后查询忘记加 is_deleted 条件 → 查到已删除数据

## 6. INSERT ON DUPLICATE KEY UPDATE（Upsert）

### 🏠 生活类比

> **Upsert** = 看看这个人在不在名单上，在就更新，不在就添加

### 📖 概念讲解

MySQL 特有的 Upsert 语法：
- 若唯一键/主键冲突 → 执行 UPDATE
- 若不冲突 → 执行 INSERT

### 💻 SQL 代码示例

```sql
USE dml_demo;

CREATE TABLE t_stock (
    product_code VARCHAR(20) PRIMARY KEY,
    product_name VARCHAR(50),
    stock INT DEFAULT 0,
    last_update DATETIME DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO t_stock (product_code, product_name, stock) VALUES
    ('P001', 'Laptop', 100),
    ('P002', 'Mouse', 500);

-- 场景1：新增库存记录
INSERT INTO t_stock (product_code, product_name, stock)
VALUES ('P003', 'Keyboard', 200)
ON DUPLICATE KEY UPDATE stock = stock + 200;

-- 场景2：更新已存在的库存（库存+50）
INSERT INTO t_stock (product_code, product_name, stock)
VALUES ('P001', 'Laptop', 50)
ON DUPLICATE KEY UPDATE
    stock = stock + VALUES(stock),
    last_update = NOW();

SELECT * FROM t_stock;

-- affected_rows 返回值判断：
-- 1 = 新插入，2 = 更新（MySQL 协议层定义）
```

### ❌ 常见错误

- ❌ Upsert 前不确认主键/唯一键 → 可能意外更新错误记录

## 本章小结

```
┌─────────────────────────────────────────────────────────┐
│                    增删改数据核心                        │
├─────────────────────────────────────────────────────────┤
│ INSERT：单行/多行/INSERT SELECT/SET/REPLACE/IGNORE      │
│   多行插入 = VALUES (...), (...), (...) 效率最高       │
│                                                          │
│ UPDATE：SET col=val WHERE condition                     │
│   ⚠️ 不带 WHERE = 更新整表！                           │
│   可回滚，ORDER BY + LIMIT 可限制更新范围              │
│                                                          │
│ DELETE：DELETE FROM t WHERE condition                   │
│   ⚠️ DELETE FROM t（无WHERE）= 清空表，但可回滚       │
│   TRUNCATE = 清空表，不可回滚，重置 AUTO_INCREMENT     │
│                                                          │
│ 批量插入：INSERT...SELECT 或 INSERT VALUES (...),()     │
│ 逻辑删除：is_deleted=1 UPDATE，保留数据可恢复            │
│ Upsert：INSERT...ON DUPLICATE KEY UPDATE               │
│   有则更新，无则插入                                     │
└─────────────────────────────────────────────────────────┘
```

**练习 3：AUTO_INCREMENT 对比**
对一张有 AUTO_INCREMENT 主键的表，分别执行 DELETE 全表、TRUNCATE 全表，之后各插入一条新记录，对比 AUTO_INCREMENT 的起始值差异。

**练习 4：INSERT IGNORE vs ON DUPLICATE KEY UPDATE**
用 INSERT IGNORE 插入一条与唯一键冲突的记录，再用 INSERT 插入冲突记录后用 ON DUPLICATE KEY UPDATE 处理，对比两者行为差异。
