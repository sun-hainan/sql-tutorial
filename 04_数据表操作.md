# 模块四：数据表操作

## 学习目标

- 掌握创建、查看、删除数据表的方法
- 掌握 ALTER TABLE 的各种用法（添加列、修改列、删除列、重命名列）
- 掌握复制表结构（LIKE）和复制表数据的方法
- 理解 TRUNCATE vs DELETE 的性能差异
- 理解临时表的概念和使用场景

## 1. 创建、查看、删除数据表

### 🏠 生活类比

> **创建表** = 在租来的写字楼里隔出一间间办公室（规定每间放什么、多少人）
> **DROP TABLE** = 拆掉整个房间（永久消失）

### 📖 概念讲解

`CREATE TABLE` 需要指定：表名、列名、列类型、约束、存储引擎、字符集。

DESC 显示列信息：Field（列名）、Type（类型）、Null（是否允许NULL）、Key（索引）、Default（默认值）、Extra（扩展）

### 💻 SQL 代码示例

```sql
CREATE DATABASE table_ops_demo;
USE table_ops_demo;

-- 创建表（完整语法）
CREATE TABLE t_employee (
    id INT PRIMARY KEY AUTO_INCREMENT,     -- 主键，自增
    name VARCHAR(50) NOT NULL,             -- 非空字符串
    email VARCHAR(100) UNIQUE,             -- 唯一约束
    salary DECIMAL(10,2) DEFAULT 0.00,     -- 默认值
    dept_id INT,                           -- 普通整型
    hiredate DATE,                         -- 日期类型
    status TINYINT DEFAULT 1               -- 小整型，默认1
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4;

-- 查看当前数据库所有表
SHOW TABLES;

-- 查看表结构
DESC t_employee;
DESCRIBE t_employee;

-- 查看建表语句（完整 DDL）
SHOW CREATE TABLE t_employee;

-- 插入测试数据
INSERT INTO t_employee (name, email, salary, dept_id, hiredate) VALUES
    ('Alice', 'alice@example.com', 8000.00, 1, '2023-01-15'),
    ('Bob', 'bob@example.com', 9000.00, 1, '2023-03-20'),
    ('Carol', 'carol@example.com', 7500.00, 2, '2023-06-01');

-- 删除表（危险！数据永久丢失）
DROP TABLE IF EXISTS t_employee;
```

### ❌ 常见错误

- ❌ 建表不指定字符集 → 默认继承数据库字符集，中文可能乱码
- ❌ DROP TABLE 误操作 → 数据永久丢失，无法恢复

## 2. ALTER TABLE 详解

### 🏠 生活类比

> **ALTER TABLE** = 装修：可以加墙（添加列）、拆墙（删除列）、改墙（修改列类型）

### 📖 概念讲解

ALTER TABLE 是修改已存在表的结构，不影响已有数据（但大表改列会有性能问题）。

| 操作 | 语法 |
|------|------|
| 添加列 | `ALTER TABLE t ADD COLUMN col TYPE ...` |
| 修改列 | `ALTER TABLE t MODIFY COLUMN col NEW_TYPE` |
| 删除列 | `ALTER TABLE t DROP COLUMN col` |
| 重命名列 | `ALTER TABLE t CHANGE COLUMN old_name new_name TYPE` |
| 添加索引 | `ALTER TABLE t ADD INDEX idx_name (col)` |
| 添加外键 | `ALTER TABLE t ADD FOREIGN KEY (col) REFERENCES other(pk)` |
| 修改表名 | `ALTER TABLE t RENAME TO new_t` |

### 💻 SQL 代码示例

```sql
USE table_ops_demo;

CREATE TABLE t_product (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);

-- 添加列（在 name 之后）
ALTER TABLE t_product ADD COLUMN price DECIMAL(10,2) AFTER name;

-- 添加列在最前面
ALTER TABLE t_product ADD COLUMN category VARCHAR(50) FIRST;

-- 添加多列
ALTER TABLE t_product
    ADD COLUMN brand VARCHAR(50),
    ADD COLUMN stock INT DEFAULT 0;

-- 修改列类型
ALTER TABLE t_product MODIFY COLUMN name VARCHAR(100);
ALTER TABLE t_product MODIFY COLUMN name VARCHAR(100) DEFAULT 'Unknown';

-- 重命名列（同时可改类型）
ALTER TABLE t_product CHANGE COLUMN brand manufacturer VARCHAR(50);

-- 删除列
ALTER TABLE t_product DROP COLUMN stock;

-- 添加索引
ALTER TABLE t_product ADD INDEX idx_category (category);
ALTER TABLE t_product ADD UNIQUE INDEX idx_name (name);

-- 添加外键（需要先有被引用表）
CREATE TABLE t_category (
    cat_id INT PRIMARY KEY,
    cat_name VARCHAR(50)
);
ALTER TABLE t_product ADD FOREIGN KEY (category) REFERENCES t_category(cat_id);

-- 修改表名
ALTER TABLE t_product RENAME TO t_goods;

SHOW TABLES;
```

### ❌ 常见错误

- ❌ ALTER TABLE 大表时直接改 → 可能锁表，用 ALGORITHM=INPLACE 减少影响
- ❌ 删除有外键引用的列 → 先删外键或删除引用表

## 3. 复制表结构与表数据

### 📖 概念讲解

| 方法 | 语法 | 复制索引 | 复制外键 | 复制数据 |
|------|------|---------|---------|---------|
| LIKE | `CREATE TABLE new LIKE old` | ✅ | ✅ | ❌ |
| AS SELECT | `CREATE TABLE new AS SELECT` | ❌ | ❌ | ✅（可选） |
| INSERT SELECT | `INSERT INTO new SELECT * FROM old` | 需先建 | 需先建 | ✅ |

### 💻 SQL 代码示例

```sql
USE table_ops_demo;

CREATE TABLE t_orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_name VARCHAR(50),
    amount DECIMAL(10,2),
    order_date DATE
);
INSERT INTO t_orders (customer_name, amount, order_date) VALUES
    ('Alice', 299.00, '2024-01-15'),
    ('Bob', 599.00, '2024-01-20'),
    ('Alice', 899.00, '2024-02-01');

-- 方法1：复制表结构（不复制数据）
CREATE TABLE t_orders_backup LIKE t_orders;
SHOW CREATE TABLE t_orders_backup;  -- 约束和索引都复制过来了
DESC t_orders_backup;
INSERT INTO t_orders_backup SELECT * FROM t_orders;  -- 再单独复制数据

-- 方法2：完整复制（含数据，不复制索引/外键）
CREATE TABLE t_orders_copy AS SELECT * FROM t_orders;
SHOW CREATE TABLE t_orders_copy;  -- 没有 AUTO_INCREMENT 等属性
DESC t_orders_copy;

-- 方法3：只复制部分列
CREATE TABLE t_orders_partial AS SELECT order_id, customer_name FROM t_orders;
DESC t_orders_partial;

-- 方法4：复制符合条件的部分数据
CREATE TABLE t_orders_alice AS SELECT * FROM t_orders WHERE customer_name = 'Alice';
```

### ❌ 常见错误

- ❌ 用 CREATE TABLE ... AS SELECT 以为复制了所有属性 → 不复制自增和索引
- ❌ 复制表后不检查约束 → 唯一约束、外键等需要手动重建

## 4. TRUNCATE vs DELETE

### 🏠 生活类比

> **DELETE** = 一个个把抽屉里的东西拿出来扔掉（慢，但可以反悔）
> **TRUNCATE** = 直接把整个抽屉扔掉换新的（快，但东西没了）

### 💻 SQL 代码示例

```sql
USE table_ops_demo;

CREATE TABLE t_truncate_test (
    id INT PRIMARY KEY AUTO_INCREMENT,
    data VARCHAR(50)
);
INSERT INTO t_truncate_test (data) VALUES ('A'), ('B'), ('C');
SELECT * FROM t_truncate_test;  -- 3条，AUTO_INCREMENT=3

-- DELETE：可回滚，不重置 AUTO_INCREMENT
START TRANSACTION;
DELETE FROM t_truncate_test;
ROLLBACK;
SELECT MAX(id) FROM t_truncate_test;  -- 仍是 3

-- TRUNCATE：不可回滚，重置 AUTO_INCREMENT
TRUNCATE TABLE t_truncate_test;
INSERT INTO t_truncate_test (data) VALUES ('X');
SELECT * FROM t_truncate_test;  -- id=1（重置了）

-- 何时用 DELETE：需要条件删除、可能需要回滚、小数据量
-- 何时用 TRUNCATE：清空测试数据、清空临时表、不需要保留数据
```

### ❌ 常见错误

- ❌ 在事务中用 TRUNCATE 以为能回滚 → TRUNCATE 是 DDL，自动提交
- ❌ TRUNCATE 有外键关联的表 → 先禁用外键检查或删除子表

## 5. 选择合适的数据类型

### 📖 概念讲解

| 类型选择 | 原则 | 示例 |
|---------|------|------|
| 整数 | 够用就行，选小不选大 | 年龄用 TINYINT（0-255） |
| CHAR vs VARCHAR | 固定长度 → CHAR，可变 → VARCHAR | 邮编 → CHAR(6)，姓名 → VARCHAR |
| DECIMAL vs FLOAT | 货币精确 → DECIMAL，科学计算 → FLOAT/DOUBLE | 价格 → DECIMAL(10,2) |

### 💻 SQL 代码示例

```sql
USE table_ops_demo;

CREATE TABLE t_data_type_selection (
    -- 整数选择
    id INT PRIMARY KEY,
    age TINYINT UNSIGNED,               -- 年龄 0-255，用 TINYINT
    user_level SMALLINT DEFAULT 1,      -- 用户等级，用 SMALLINT

    -- 字符串选择
    code CHAR(6) NOT NULL,              -- 固定6位编码，如 ABC001
    phone VARCHAR(20),                   -- 可变长度手机号
    bio TEXT,                           -- 个人简介，可能很长

    -- 数值选择
    price DECIMAL(10,2),                -- 商品价格，必须精确
    latitude DOUBLE,                     -- 经纬度，DOUBLE 足够
    ratio FLOAT,                         -- 比例，近似值即可

    -- 日期时间
    birth_date DATE,
    create_time DATETIME
);

-- CHAR vs VARCHAR 空间占用对比
CREATE TABLE t_string_compare (
    fixed CHAR(10),       -- 固定占10字节，不足用空格填充
    variable VARCHAR(10)  -- 只占实际长度+1~2字节前缀
);
INSERT INTO t_string_compare VALUES ('ABC', 'ABC');
SELECT LENGTH(fixed), LENGTH(variable) FROM t_string_compare;
-- CHAR(10)：10 字节（固定长度）
-- VARCHAR(10)：3 字节 + 长度前缀
```

### ❌ 常见错误

- ❌ 用 FLOAT 存金额 → 精度丢失，应该用 DECIMAL
- ❌ VARCHAR 不设最大长度 → 最大 65535 字节，需评估业务最大需求

## 6. 临时表

### 🏠 生活类比

> **临时表** = 纸上的草稿（当前会话有效，断开就扔掉）

### 📖 概念讲解

- `CREATE TEMPORARY TABLE`：只在当前连接存在，断开连接后自动删除
- 可以与普通表同名（当前会话优先使用临时表）
- 不支持外键
- 适合复杂查询的中间结果、临时的数据处理

### 💻 SQL 代码示例

```sql
USE table_ops_demo;

-- 创建临时表
CREATE TEMPORARY TABLE t_temp_monthly_sales AS
SELECT
    MONTH(order_date) AS month_num,
    SUM(amount) AS total_sales
FROM t_orders
GROUP BY MONTH(order_date);

SELECT * FROM t_temp_monthly_sales;

-- 临时表与普通表同名时，当前会话优先使用临时表
CREATE TEMPORARY TABLE t_orders AS SELECT * FROM t_orders WHERE 1=0;
-- 此时 t_orders 指的是临时表，不是真实的订单表

-- 临时表用于数据迁移测试
CREATE TEMPORARY TABLE t_orders_staging AS
SELECT * FROM t_orders LIMIT 0;  -- 只复制结构

INSERT INTO t_orders_staging SELECT * FROM t_orders WHERE customer_name = 'Alice';

-- 查看临时表
SHOW TEMPORARY TABLES;
```

### ❌ 常见错误

- ❌ 以为临时表数据会持久化 → 断开连接后自动删除
- ❌ 临时表与普通表同名时混淆 → 建议临时表加 `tmp_` 前缀

## 7. 存储引擎选择

### 📖 概念讲解

**MySQL 5.5+ 默认 InnoDB**，除非有特殊需求（如全文索引）才用 MyISAM。

| 场景 | 推荐引擎 |
|------|---------|
| 通用场景/OLTP | InnoDB ✅ |
| 读多写少/静态数据 | MyISAM |
| 需要 FULLTEXT 索引 | MyISAM（或 InnoDB 5.6+） |
| 内存表/缓存 | MEMORY |

### 💻 SQL 代码示例

```sql
USE table_ops_demo;

-- InnoDB 表（默认）
CREATE TABLE t_innodb_demo (
    id INT PRIMARY KEY,
    name VARCHAR(50)
) ENGINE = InnoDB;

-- MyISAM 表（需要全文索引时）
CREATE TABLE t_myisam_demo (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    FULLTEXT INDEX ft_name (name)  -- MyISAM 支持全文索引
) ENGINE = MyISAM;

INSERT INTO t_innodb_demo VALUES (1, 'InnoDB特性：事务+行锁');
INSERT INTO t_myisam_demo VALUES (1, 'MyISAM特性：支持FULLTEXT');

-- 查看表存储引擎
SHOW TABLE STATUS LIKE 't_%';
```

### ❌ 常见错误

- ❌ 所有表都用 MyISAM → 不支持事务和外键，并发写入差
