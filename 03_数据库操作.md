# 模块三：数据库操作

## 学习目标

- 掌握创建、切换、删除数据库的方法
- 理解字符集（CHARACTER SET）和排序规则（COLLATION）的概念
- 掌握通过 INFORMATION_SCHEMA 查询数据库元数据
- 理解 MySQL 存储引擎的概念及 InnoDB vs MyISAM 区别
- 掌握 DROP DATABASE / TRUNCATE TABLE / DROP TABLE 的区别

---

## 1. 创建、切换、删除数据库

### 🏠 生活类比

> **创建数据库** = 租一间写字楼（给你分配空间）
> **切换数据库** = 进入这间楼的某一层（USE 命令）
> **删除数据库** = 退租整栋楼（所有东西都没了，不可恢复）

### 📖 概念讲解

数据库是 MySQL 中最大级别的命名空间，用于隔离不同项目的数据。

### 💻 SQL 代码示例

```sql
-- 创建数据库（指定字符集和排序规则，推荐 utf8mb4）
CREATE DATABASE IF NOT EXISTS shop_db
    DEFAULT CHARACTER SET utf8mb4          -- 完整 UTF-8，支持 emoji
    DEFAULT COLLATE utf8mb4_unicode_ci;    -- 通用排序规则，区分大小写

-- 切换数据库
USE shop_db;

-- 在当前数据库中创建表
CREATE TABLE t_product (
    id INT PRIMARY KEY,
    name VARCHAR(100)
);

-- 删除数据库（危险！所有表和数据永久删除）
DROP DATABASE IF EXISTS shop_db;
```

### ❌ 常见错误

- ❌ 不指定字符集 → MySQL 默认 latin1，不支持中文
- ❌ 删除前不备份 → 数据永久丢失，无法恢复

## 2. 字符集与排序规则

### 🏠 生活类比

> **字符集** = 字典里收录了哪些汉字（GBK 只收录简体，utf8mb4 收录所有文字）
> **排序规则** = 比较汉字大小的规则（按拼音？按笔画？按 Unicode 码点？）

### 📖 概念讲解

| 字符集 | 说明 | 中文支持 |
|--------|------|---------|
| latin1 | 西欧字符 | ❌ 不支持 |
| gbk | 简体中文 | ✅ |
| utf8 | MySQL 早期 3 字节 UTF-8 | ✅ 但缺 emoji |
| **utf8mb4** | 完整 4 字节 UTF-8 | ✅ 推荐 |

| 排序规则 | 说明 |
|----------|------|
| utf8mb4_general_ci | 通用排序，不区分大小写，性能优先 |
| utf8mb4_unicode_ci | Unicode 标准排序，精确度高 |
| utf8mb4_bin | 二进制比较，区分大小写 |

### 💻 SQL 代码示例

```sql
CREATE DATABASE charset_demo
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;

USE charset_demo;

CREATE TABLE t_text_test (
    id INT PRIMARY KEY,
    content VARCHAR(100)
);

INSERT INTO t_text_test VALUES (1, 'Apple');
INSERT INTO t_text_test VALUES (2, 'apple');
INSERT INTO t_text_test VALUES (3, 'APPLE');

-- 排序规则影响查询（不区分大小写）
SELECT * FROM t_text_test WHERE content = 'apple';
-- utf8mb4_unicode_ci：找到 1 条（apple）
-- utf8mb4_bin：找到 1 条（apple，严格区分大小写）

-- 演示区分大小写
CREATE TABLE t_case_test (
    id INT PRIMARY KEY,
    name VARCHAR(50) COLLATE utf8mb4_bin  -- 列级别指定排序规则
);
INSERT INTO t_case_test VALUES (1, 'abc'), (2, 'ABC'), (3, 'Abc');

SELECT * FROM t_case_test WHERE name = 'abc';  -- 只返回 'abc'
```

### ❌ 常见错误

- ❌ 数据库用 utf8，表用 gbk → 字符集混乱，应该统一用 utf8mb4
- ❌ 排序规则混用 → 同一字段比较时排序规则不一致会报错

## 3. 查看数据库属性

### 📖 概念讲解

| 命令 | 作用 |
|------|------|
| SHOW DATABASES | 列出所有数据库 |
| SHOW CREATE DATABASE | 查看创建语句（含字符集） |
| SHOW TABLE STATUS | 查看表信息 |
| SHOW TABLES | 列出当前数据库所有表 |
| DESC / DESCRIBE | 查看表结构 |
| INFORMATION_SCHEMA | 系统数据库，存储元数据 |

### 💻 SQL 代码示例

```sql
-- 列出所有数据库
SHOW DATABASES;

-- 查看某数据库的创建信息
SHOW CREATE DATABASE mysql;

-- 查看当前数据库所有表
SHOW TABLES;

-- 查看某表的创建语句
SHOW CREATE TABLE mysql.user;

-- 通过 INFORMATION_SCHEMA 查询数据库信息
SELECT
    SCHEMA_NAME,
    DEFAULT_CHARACTER_SET_NAME,
    DEFAULT_COLLATION_NAME
FROM INFORMATION_SCHEMA.SCHEMATA;

-- 查看某表的所有列信息
SELECT
    COLUMN_NAME,
    DATA_TYPE,
    CHARACTER_SET_NAME,
    COLLATION_NAME
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = 'mysql'
  AND TABLE_NAME = 'user'
LIMIT 10;
```

### ❌ 常见错误

- ❌ 直接操作 mysql 系统库 → 可能破坏数据库，应避免直接修改

## 4. MySQL 存储引擎

### 🏠 生活类比

> **存储引擎**就像不同的仓库管理系统：
> - **InnoDB** = 现代化仓库（有货架、叉车、门禁，支持货物追踪）
> - **MyISAM** = 老式仓库（简单货架，不支持复杂管理）

### 📖 概念讲解

| 特性 | InnoDB | MyISAM |
|------|--------|--------|
| 事务支持 | ✅ 支持 | ❌ 不支持 |
| 锁级别 | 行级锁 | 表级锁 |
| 外键 | ✅ 支持 | ❌ 不支持 |
| 全文索引 | 5.6+ 支持 | ✅ 原生支持 |
| 崩溃恢复 | 自动恢复 | 需手动修复 |
| 适用场景 | 写多/并发/事务 | 读多/静态/日志 |
| 存储空间 | 约 2 倍 | 约 1 倍 |

### 💻 SQL 代码示例

```sql
-- 查看支持的存储引擎
SHOW ENGINES;

-- 创建 InnoDB 表（MySQL 5.5+ 默认引擎）
CREATE TABLE t_innodb (
    id INT PRIMARY KEY,
    name VARCHAR(50)
) ENGINE = InnoDB;

-- 创建 MyISAM 表
CREATE TABLE t_myisam (
    id INT PRIMARY KEY,
    name VARCHAR(50)
) ENGINE = MyISAM;

-- 插入相同数据
INSERT INTO t_innodb VALUES (1, 'Alice'), (2, 'Bob');
INSERT INTO t_myisam VALUES (1, 'Alice'), (2, 'Bob');

-- 查看存储引擎状态
SHOW TABLE STATUS FROM mysql LIKE 'user';
```

### ❌ 常见错误

- ❌ 需要事务还用 MyISAM → 数据一致性无法保证
- ❌ 混用存储引擎 → 不同引擎之间无法外键关联

## 5. 复制与重命名数据库

### 📖 概念讲解

MySQL 没有直接命令复制整个数据库，需要使用 **mysqldump** 工具导出再导入。

### 💻 SQL 代码示例

```sql
-- 重命名表（在同一数据库内）
USE shop_db;
CREATE TABLE t_test_rename (id INT);
RENAME TABLE t_test_rename TO t_renamed;

SHOW TABLES;

-- 复制数据库（需要 mysqldump 工具）：
-- 步骤1：导出
-- mysqldump -u root -p shop_db > shop_db_backup.sql
-- 步骤2：创建新数据库
-- CREATE DATABASE shop_db_backup;
-- 步骤3：导入数据
-- mysql -u root -p shop_db_backup < shop_db_backup.sql
```

### ❌ 常见错误

- ❌ 用 SELECT INTO 创建跨库表 → MySQL 禁止，需要用 INSERT SELECT

## 6. 默认数据库与跨库查询

### 📖 概念讲解

| 概念 | 说明 |
|------|------|
| 默认数据库 | USE 选择后，当前会话的默认数据库 |
| 完全限定名 | `database_name.table_name.column_name`，明确指定 |

### 💻 SQL 代码示例

```sql
DROP DATABASE IF EXISTS db1;
DROP DATABASE IF EXISTS db2;
CREATE DATABASE db1;
CREATE DATABASE db2;

USE db1;
CREATE TABLE t_user (id INT PRIMARY KEY, name VARCHAR(50));
INSERT INTO t_user VALUES (1, 'Alice');

USE db2;
CREATE TABLE t_order (id INT PRIMARY KEY, amount DECIMAL(10,2));
INSERT INTO t_order VALUES (1, 299.00);

-- USE db1 后，两种查询等价
USE db1;
SELECT * FROM t_user;              -- 默认在 db1 中查找
SELECT * FROM db1.t_user;          -- 完全限定名

-- 跨库查询（必须使用完全限定名）
SELECT u.name, o.amount
FROM db1.t_user u
INNER JOIN db2.t_order o ON u.id = 1;
```

### ❌ 常见错误

- ❌ 跨库查询不写完全限定名 → 报错找不到表

## 7. MySQL 配置文件

### 📖 概念讲解

| 系统 | 配置文件位置 |
|------|------------|
| Windows | my.ini（安装目录或 C:\ProgramData\MySQL） |
| Linux/Mac | my.cnf（/etc/my.cnf） |

### 💻 SQL 代码示例

```sql
-- 查看当前字符集配置
SHOW VARIABLES LIKE 'character_set%';
SHOW VARIABLES LIKE 'collation%';

-- 查看默认存储引擎
SHOW VARIABLES LIKE 'default_storage_engine';

-- 查看最大连接数
SHOW VARIABLES LIKE 'max_connections';

-- 动态修改（当前会话生效）
SET NAMES utf8mb4;

-- 动态修改（需 SUPER 权限，重启后失效）
SET GLOBAL max_connections = 500;
```

### ❌ 常见错误

- ❌ 只改配置文件不重启 → 配置不生效
- ❌ 用 SET 改配置以为永久生效 → 需要改配置文件

## 8. DROP / TRUNCATE / DELETE 三角对比

### 📖 概念讲解

**危险程度排序**：`DROP DATABASE > DROP TABLE > TRUNCATE TABLE > DELETE`

| 特性 | DROP DATABASE | DROP TABLE | TRUNCATE TABLE | DELETE |
|------|--------------|------------|---------------|--------|
| 删除内容 | 数据库+表+数据 | 表+数据 | 数据 | 数据 |
| 保留结构 | ❌ | ❌ | ✅ | ✅ |
| 类型 | DDL | DDL | DDL | DML |
| 可回滚 | ❌ | ❌ | ❌ | ✅ |
| 重置 AUTO_INCREMENT | ❌ | ❌ | ✅ | ❌ |

### 💻 SQL 代码示例

```sql
USE charset_demo;

CREATE TABLE t_del_test (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50)
);
INSERT INTO t_del_test (name) VALUES ('A'), ('B'), ('C');
SELECT * FROM t_del_test;  -- 3条，AUTO_INCREMENT=3

-- DELETE：可回滚，不重置 AUTO_INCREMENT
START TRANSACTION;
DELETE FROM t_del_test;
ROLLBACK;
SELECT * FROM t_del_test;  -- 数据恢复，AUTO_INCREMENT 仍为 3

-- TRUNCATE：不可回滚，重置 AUTO_INCREMENT
TRUNCATE TABLE t_del_test;
INSERT INTO t_del_test (name) VALUES ('X');
SELECT * FROM t_del_test;  -- id=1（AUTO_INCREMENT 重置为 1）

-- DROP TABLE：删除表结构
DROP TABLE t_del_test;
-- 表不存在了，所有数据永久丢失
```

### ❌ 常见错误

- ❌ DROP DATABASE 没备份 → 不可恢复
- ❌ DELETE 全表后以为能回滚 → 如果不在事务中则无法回滚


**练习 1：创建带字符集的数据库**
创建一个名为 `test_charset` 的数据库，指定 utf8mb4 字符集和 utf8mb4_unicode_ci 排序规则，然后创建表插入中英文混合数据，验证不区分大小写查询。

**练习 2：数据库备份与恢复**

**练习 3：DELETE / TRUNCATE / DROP 对比**
分别用 DELETE、TRUNCATE、DROP 删除同一个表的数据，用 ROLLBACK 验证哪些可以回滚，并验证 AUTO_INCREMENT 的差异。

**练习 4：查询 INFORMATION_SCHEMA**
查询 `INFORMATION_SCHEMA.SCHEMATA`，找出 MySQL 默认的字符集和排序规则，列出所有可用字符集。

**练习 5：临时表可见性**
创建一个临时表，验证 SHOW TABLES 和 `SHOW FULL TABLES WHERE Table_type = 'VIEW'` 对临时表的可见性差异。
