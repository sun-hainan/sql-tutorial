# 模块六：六大约束

## 学习目标

- 掌握主键约束（PRIMARY KEY）的概念和联合主键
- 理解唯一约束（UNIQUE）与主键约束的区别
- 掌握非空约束（NOT NULL）和默认值约束（DEFAULT）
- 掌握检查约束（CHECK）的用法（MySQL 8.0.16+）
- 掌握外键约束（FOREIGN KEY）的四个选项（CASCADE/SET NULL/RESTRICT/NO ACTION）

---

## 1. 主键约束（PRIMARY KEY）

### 🏠 生活类比

> **主键** = 人的身份证号——唯一标识每一个人，**不能重复，不能为空**。

### 📖 概念讲解

| 特性 | 说明 |
|------|------|
| 唯一性 | 主键值唯一，不重复 |
| 非空 | 主键值不能为 NULL |
| 数量 | 每表只能有一个主键 |
| 索引 | 自动创建唯一索引（InnoDB 是聚簇索引） |
| 类型 | 单列主键 或 联合主键（多列组合唯一） |

### 💻 SQL 代码示例

```sql
CREATE DATABASE constraints_demo;
USE constraints_demo;

-- 单列主键
CREATE TABLE t_user_single_pk (
    user_id INT PRIMARY KEY,       -- 主键：唯一且非空
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100)
);
INSERT INTO t_user_single_pk VALUES (1, 'alice', 'alice@example.com');
-- 重复主键会报错
-- INSERT INTO t_user_single_pk VALUES (1, 'bob', 'bob@example.com');  -- Error
-- NULL 主键会报错
-- INSERT INTO t_user_single_pk VALUES (NULL, 'charlie', 'charlie@example.com');  -- Error

-- 自增主键（AUTO_INCREMENT 配合主键使用）
CREATE TABLE t_order (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_name VARCHAR(50),
    amount DECIMAL(10,2)
);
INSERT INTO t_order (customer_name, amount) VALUES ('Alice', 299.00);
INSERT INTO t_order (customer_name, amount) VALUES ('Bob', 599.00);
SELECT * FROM t_order;  -- order_id 自动为 1, 2, 3...

-- 联合主键（复合主键）：student_id + course_id 组合唯一
CREATE TABLE t_enrollment (
    student_id INT,
    course_id INT,
    enroll_date DATE,
    grade DECIMAL(4,1),
    PRIMARY KEY (student_id, course_id)  -- 联合主键
);
INSERT INTO t_enrollment VALUES (1, 101, '2024-01-01', 85.5);
INSERT INTO t_enrollment VALUES (1, 102, '2024-01-02', 90.0);
INSERT INTO t_enrollment VALUES (2, 101, '2024-01-01', 78.0);
-- 同一学生重复选修同一课程会报错
-- INSERT INTO t_enrollment VALUES (1, 101, '2024-02-01', 88.0);  -- Error
```

### 📊 ASCII 图

```
┌─────────────────────────────────────────────────────┐
│                   主键约束                           │
├─────────────────────────────────────────────────────┤
│ t_enrollment（选课表）                              │
│ ┌───────────┬───────────┬────────────┬────────┐    │
│ │ student_id│ course_id │ enroll_date │ grade  │    │
│ │  (PK)     │   (PK)   │            │        │    │
│ ├───────────┼───────────┼────────────┼────────┤    │
│ │    1      │   101    │ 2024-01-01 │  85.5  │ ← ① │
│ │    1      │   102    │ 2024-01-02 │  90.0  │ ← ② │
│ │    2      │   101    │ 2024-01-01 │  78.0  │ ← ③ │
│ └───────────┴───────────┴────────────┴────────┘    │
│ PK = (student_id, course_id) 组合唯一             │
└─────────────────────────────────────────────────────┘
```

### ❌ 常见错误

- ❌ 用业务字段（邮箱、用户名）做主键 → 业务可能变化，建议用自增 ID
- ❌ 忘记给主键建索引 → InnoDB 自动创建，不需要手动建

## 2. 唯一约束（UNIQUE）与主键的区别

### 🏠 生活类比

> **主键** = 身份证号（必须唯一且非空，一个人只能有一个）
> **唯一约束** = 护照号（可以为空，但不是每个人都有，且可以有多个人都没有护照）

### 📖 概念讲解

| 特性 | 主键（PRIMARY KEY） | 唯一约束（UNIQUE） |
|------|---------------------|------------------|
| 数量 | 每表只能一个 | 每表可以有多个 |
| NULL | 不允许 NULL | 允许单个 NULL（多个 NULL 也不冲突） |
| 索引类型 | 聚簇索引 | 辅助索引 |
| 用途 | 唯一标识记录 | 确保某列值不重复 |

### 💻 SQL 代码示例

```sql
USE constraints_demo;

-- 主键 + 唯一约束组合
CREATE TABLE t_employee (
    emp_id INT PRIMARY KEY,           -- 主键：唯一且非空
    emp_code VARCHAR(20) UNIQUE,      -- 唯一约束：工号唯一
    emp_name VARCHAR(50)
);
INSERT INTO t_employee VALUES (1, 'E001', 'Alice');
INSERT INTO t_employee VALUES (2, 'E002', 'Bob');
-- emp_id 重复报错，emp_code 重复也报错

-- NULL 的情况
CREATE TABLE t_unique_null (
    id INT PRIMARY KEY,
    code VARCHAR(20) UNIQUE
);
INSERT INTO t_unique_null VALUES (1, NULL);   -- OK
INSERT INTO t_unique_null VALUES (2, NULL);   -- OK：多个 NULL 互不冲突
SELECT * FROM t_unique_null;  -- 两条记录，code 都是 NULL
```

### ❌ 常见错误

- ❌ 混淆主键和唯一约束 → 主键不能为空，唯一约束可以

## 3. 非空约束（NOT NULL）和默认值（DEFAULT）

### 🏠 生活类比

> **NOT NULL** = 必填项（身份证号必须填）
> **DEFAULT** = 预填项（表格上已经印好的默认选项）

### 💻 SQL 代码示例

```sql
USE constraints_demo;

-- NOT NULL 约束
CREATE TABLE t_member (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,    -- 必须提供用户名
    email VARCHAR(100) NOT NULL,      -- 必须提供邮箱
    phone VARCHAR(20) NULL,            -- 可选填
    age INT                           -- 可选，默认 NULL
);
INSERT INTO t_member (username, email, phone, age) VALUES ('alice', 'alice@example.com', '13800138000', 25);
INSERT INTO t_member (username, email) VALUES ('bob', 'bob@example.com');  -- phone 和 age 可省略
-- INSERT INTO t_member (username) VALUES ('charlie');  -- Error: email cannot be NULL

-- DEFAULT 默认值
CREATE TABLE t_post (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    author_id INT NOT NULL,
    status VARCHAR(20) DEFAULT 'draft',    -- 默认'草稿'状态
    views INT DEFAULT 0,                    -- 默认0浏览量
    is_deleted TINYINT DEFAULT 0,          -- 默认未删除
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
INSERT INTO t_post (title, content, author_id) VALUES ('My First Post', 'Hello world', 1);
-- status='draft', views=0, is_deleted=0

-- DEFAULT 值可以是表达式
CREATE TABLE t_invoice (
    id INT PRIMARY KEY AUTO_INCREMENT,
    invoice_no VARCHAR(50) DEFAULT (CONCAT('INV', YEAR(NOW()), LPAD(id, 6, '0'))),
    amount DECIMAL(10,2) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
INSERT INTO t_invoice (amount) VALUES (1000.00);
SELECT * FROM t_invoice;  -- invoice_no 自动生成
```

### ❌ 常见错误

- ❌ NOT NULL 列不加 DEFAULT → 每次插入必须显式提供值

## 4. 检查约束（CHECK）

### 🏠 生活类比

> **CHECK** = 填表时的格式验证（年龄必须是 0-150 之间，不符合不让填）

### 📖 概念讲解

- MySQL 8.0.16+ 正式支持 CHECK 约束
- MySQL 8.0.16 之前：CHECK 被忽略（不报错但不强制）
- CHECK 在数据库层面强制，比应用层更安全

### 💻 SQL 代码示例

```sql
USE constraints_demo;

-- CHECK 约束（MySQL 8.0.16+）
CREATE TABLE t_product_check (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    price DECIMAL(10,2) NOT NULL CHECK (price > 0),              -- 价格必须>0
    quantity INT NOT NULL CHECK (quantity >= 0),                  -- 数量非负
    rating DECIMAL(2,1) CHECK (rating >= 1.0 AND rating <= 5.0)    -- 评分1-5
);
INSERT INTO t_product_check (name, price, quantity, rating) VALUES ('Laptop', 5000.00, 10, 4.5);
-- INSERT INTO t_product_check (name, price, quantity, rating) VALUES ('Book', -10.00, 5, 4.0);  -- Error

-- CHECK 约束命名（便于维护）
CREATE TABLE t_order_check (
    id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT NOT NULL,
    total_amount DECIMAL(10,2),
    paid_amount DECIMAL(10,2),
    CONSTRAINT chk_paid_positive CHECK (paid_amount >= 0),
    CONSTRAINT chk_paid_not_exceed CHECK (paid_amount <= total_amount)
);
INSERT INTO t_order_check (customer_id, total_amount, paid_amount) VALUES (1, 1000.00, 500.00);
-- INSERT INTO t_order_check VALUES (2, 1000.00, 1200.00);  -- Error: paid_amount > total_amount
```

### ❌ 常见错误

- ❌ 在 MySQL 5.7 环境中用 CHECK → 不报错但不强制，应该用触发器代替

## 5. 外键约束（FOREIGN KEY）

### 🏠 生活类比

> **外键** = 学生证上的学校编号——确保学生属于真实存在的学校，不存在就报错。

### 📖 概念讲解

子表的外键列的值必须在父表的主键中存在（或为 NULL）。

| 选项 | 说明 |
|------|------|
| CASCADE | 父删除/更新 → 子跟着删除/更新 |
| SET NULL | 父删除 → 子的外键设为 NULL |
| RESTRICT | 有子记录 → 拒绝删除父记录 |
| NO ACTION | 同 RESTRICT（MySQL 中等效） |

### 💻 SQL 代码示例

```sql
USE constraints_demo;

-- 父表：部门
CREATE TABLE t_dept (
    dept_id INT PRIMARY KEY AUTO_INCREMENT,
    dept_name VARCHAR(50) NOT NULL
);

-- 子表：员工（通过外键关联部门）
CREATE TABLE t_emp (
    emp_id INT PRIMARY KEY AUTO_INCREMENT,
    emp_name VARCHAR(50) NOT NULL,
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES t_dept(dept_id)
        ON DELETE SET NULL        -- 部门删除时，员工dept_id设为NULL
        ON UPDATE CASCADE         -- 部门ID更新时，员工dept_id同步更新
);

INSERT INTO t_dept VALUES (1, 'IT'), (2, 'HR');
INSERT INTO t_emp VALUES (1, 'Alice', 1), (2, 'Bob', 1), (3, 'Carol', 2);

-- 外键约束验证
-- INSERT INTO t_emp VALUES (4, 'David', 99);  -- Error: dept_id=99 不存在
```

### 📊 ASCII 图

```
┌─────────────────┐          ┌─────────────────┐
│    t_dept       │          │     t_emp        │
│   （父表）       │          │    （子表）       │
├─────────────────┤          ├─────────────────┤
│ dept_id (PK)    │◄────────│  dept_id (FK)   │
│ dept_name       │  外键    │  emp_id (PK)    │
└─────────────────┘  引用    │  emp_name       │
  dept_id=1: IT                 └─────────────────┘
  dept_id=2: HR
```

### ❌ 常见错误

- ❌ 外键不建索引 → 大数据量时关联查询性能差
- ❌ ON DELETE CASCADE 滥用 → 删父记录时子记录被级联删除，无法恢复

## 6. 外键 ON DELETE / ON UPDATE 选项详解

### 💻 SQL 代码示例

```sql
USE constraints_demo;

-- 父表
CREATE TABLE t_parent (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);

-- 子表1：CASCADE（级联删除/更新）
CREATE TABLE t_child_cascade (
    id INT PRIMARY KEY AUTO_INCREMENT,
    parent_id INT,
    data VARCHAR(50),
    FOREIGN KEY (parent_id) REFERENCES t_parent(id)
        ON DELETE CASCADE ON UPDATE CASCADE
);

-- 子表2：SET NULL
CREATE TABLE t_child_setnull (
    id INT PRIMARY KEY AUTO_INCREMENT,
    parent_id INT,
    data VARCHAR(50),
    FOREIGN KEY (parent_id) REFERENCES t_parent(id)
        ON DELETE SET NULL ON UPDATE SET NULL
);

-- 子表3：RESTRICT（拒绝删除有子记录的父记录）
CREATE TABLE t_child_restrict (
    id INT PRIMARY KEY AUTO_INCREMENT,
    parent_id INT,
    data VARCHAR(50),
    FOREIGN KEY (parent_id) REFERENCES t_parent(id)
        ON DELETE RESTRICT ON UPDATE RESTRICT
);

INSERT INTO t_parent VALUES (1, 'Parent1'), (2, 'Parent2');
INSERT INTO t_child_cascade VALUES (1, 1, 'C1');
INSERT INTO t_child_setnull VALUES (1, 1, 'S1');
INSERT INTO t_child_restrict VALUES (1, 1, 'R1');

-- CASCADE：删除父记录1，子记录也被删除
DELETE FROM t_parent WHERE id = 1;
SELECT * FROM t_child_cascade;  -- 空，级联删除

-- SET NULL：删除父记录2，子记录被设为 NULL
DELETE FROM t_parent WHERE id = 2;
SELECT * FROM t_child_setnull;  -- parent_id=NULL

-- RESTRICT：阻止删除有子记录的父记录
-- DELETE FROM t_parent WHERE id = 1;  -- Error: Cannot delete
```

### ❌ 常见错误

- ❌ ON DELETE CASCADE 不小心用错 → 数据被级联删除，无法恢复

## 7. 查看约束和索引

### 💻 SQL 代码示例

```sql
USE constraints_demo;

CREATE TABLE t_constraint_demo (
    id INT PRIMARY KEY AUTO_INCREMENT,
    code VARCHAR(20) NOT NULL UNIQUE,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100),
    dept_id INT,
    age INT CHECK (age >= 0 AND age <= 150),
    CONSTRAINT fk_dept FOREIGN KEY (dept_id) REFERENCES t_dept(id),
    CONSTRAINT uk_code_name UNIQUE (code, name)
);

-- 方法1：SHOW CREATE TABLE
SHOW CREATE TABLE t_constraint_demo;

-- 方法2：SHOW INDEX
SHOW INDEX FROM t_constraint_demo;

-- 方法3：INFORMATION_SCHEMA 查询约束
SELECT
    CONSTRAINT_NAME,
    TABLE_NAME,
    CONSTRAINT_TYPE
FROM INFORMATION_SCHEMA.TABLE_CONSTRAINTS
WHERE TABLE_SCHEMA = 'constraints_demo'
  AND TABLE_NAME = 't_constraint_demo';

-- 方法4：INFORMATION_SCHEMA 查询索引
SELECT
    INDEX_NAME,
    TABLE_NAME,
    COLUMN_NAME,
    NON_UNIQUE
FROM INFORMATION_SCHEMA.STATISTICS
WHERE TABLE_SCHEMA = 'constraints_demo'
  AND TABLE_NAME = 't_constraint_demo';
```

### ❌ 常见错误

- ❌ 用 DESC 代替 SHOW CREATE TABLE → DESC 不显示约束详情

## 本章小结

```
┌─────────────────────────────────────────────────────────┐
│                    六大约束                              │
├─────────────────────────────────────────────────────────┤
│ PRIMARY KEY → 唯一非空，每表一个，聚簇索引               │
│ UNIQUE → 唯一，可NULL，每表多个，辅助索引                │
│ NOT NULL → 字段值不能为空                               │
│ DEFAULT → 插入时不提供值则使用默认值                     │
│ CHECK → 字段值必须满足条件（MySQL 8.0.16+支持）         │
│ FOREIGN KEY → 确保引用完整性，ON DELETE/UPDATE 选项     │
│                                                          │
│ 外键选项：                                               │
│   CASCADE → 父删/更新，子跟着删/更新（最常用）          │
│   SET NULL → 父删除,子外键设为NULL                      │
│   RESTRICT → 有子记录则拒绝删除父记录                    │
│                                                          │
│ NOT NULL vs DEFAULT：                                   │
│   NOT NULL → 不提供值就报错                              │
│   DEFAULT → 不提供值则使用默认值                        │
└─────────────────────────────────────────────────────────┘
```
