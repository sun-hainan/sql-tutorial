# 模块五：数据类型

## 学习目标

- 掌握数值类型（TINYINT / INT / BIGINT / DECIMAL / FLOAT / DOUBLE）
- 掌握字符串类型（CHAR / VARCHAR / TEXT）的特点和选择原则
- 掌握日期时间类型（DATE / DATETIME / TIMESTAMP）的区别
- 理解 ENUM 和 SET 的使用场景
- 理解 DECIMAL vs FLOAT 的精度问题

---

## 1. 数值类型

### 🏠 生活类比

> **整数类型**就像不同的容器：TINYINT 是一个小盒子（放年龄），BIGINT 是一个大仓库（放天文数字）

### 📖 概念讲解

| 类型 | 字节 | 有符号范围 | 无符号范围 |
|------|------|-----------|-----------|
| TINYINT | 1 | -128 ~ 127 | 0 ~ 255 |
| SMALLINT | 2 | -32768 ~ 32767 | 0 ~ 65535 |
| MEDIUMINT | 3 | -8388608 ~ 8388607 | 0 ~ 16777215 |
| INT | 4 | -21亿 ~ 21亿 | 0 ~ 42亿 |
| BIGINT | 8 | 极大 | 0 ~ 极大 |
| DECIMAL(M,D) | 变长 | 精确小数 | 精确小数 |
| FLOAT | 4 | 近似值 | 近似值 |
| DOUBLE | 8 | 近似值 | 近似值 |

### 💻 SQL 代码示例

```sql
CREATE DATABASE data_types_demo;
USE data_types_demo;

CREATE TABLE t_numeric_types (
    -- 有符号整数
    col_tinyint_signed TINYINT,
    col_smallint SIGNED,
    col_mediumint MEDIUMINT,
    col_int INT,
    col_bigint BIGINT,

    -- 无符号整数（只有正数）
    col_unsigned TINYINT UNSIGNED,
    col_positive SMALLINT UNSIGNED,

    -- 浮点类型
    col_float FLOAT,
    col_double DOUBLE,
    col_decimal DECIMAL(10,2),   -- 最多10位数字，2位小数
    col_decimal2 DECIMAL(5,3),  -- 最多5位数字，3位小数

    -- BIT 类型（位字段）
    col_bit BIT(8)
);

INSERT INTO t_numeric_types VALUES (
    127,                    -- tinyint signed 最大值
    32767,                  -- smallint 最大值
    8388607,                -- mediumint 最大值
    2147483647,             -- int 最大值
    9223372036854775807,    -- bigint 最大值
    255,                    -- tinyint unsigned 最大值
    65535,                  -- smallint unsigned 最大值
    3.14159265358979,       -- float（近似值）
    3.14159265358979,       -- double（近似值）
    12345678.90,            -- decimal(10,2)
    123.456,                -- decimal(5,3)
    255                      -- bit(8)
);

SELECT
    col_tinyint_signed AS tinyint_signed,
    col_unsigned AS tinyint_unsigned,
    col_decimal,
    col_decimal2,
    col_bit + 0 AS bit_as_int  -- BIT 转数字显示
FROM t_numeric_types;
```

### ❌ 常见错误

- ❌ 所有整数都用 BIGINT → 浪费存储空间，应该选够用的最小类型
- ❌ 用 FLOAT 存金额 → 精度丢失，金额必须用 DECIMAL

## 2. 字符串类型

### 🏠 生活类比

> **CHAR(6)** = 固定大小的信封（放6个字符，不足用空格填满）
> **VARCHAR(50)** = 弹性信封（只装实际内容，省空间但多一个长度前缀）

### 📖 概念讲解

| 类型 | 最大长度 | 存储方式 |
|------|---------|---------|
| CHAR(n) | 255 | 固定 n 字节，不足用空格填充 |
| VARCHAR(n) | 65535 字节 | 实际长度 + 1~2 字节长度前缀 |
| TINYTEXT | 255 字节 | 同 VARCHAR 但无默认值 |
| TEXT | 65535 字节 | 大文本，无默认值 |
| MEDIUMTEXT | 16MB | 中等文本 |
| LONGTEXT | 4GB | 超大文本 |

### 💻 SQL 代码示例

```sql
USE data_types_demo;

CREATE TABLE t_string_types (
    id INT PRIMARY KEY AUTO_INCREMENT,

    -- CHAR：固定长度
    country_code CHAR(3),           -- 3位国家码：CHN、USA
    gender CHAR(1),                 -- M/F
    md5_hash CHAR(32),             -- MD5 固定32字符

    -- VARCHAR：可变长度
    username VARCHAR(50),
    email VARCHAR(100),
    address VARCHAR(200),

    -- TEXT 系列
    remark TEXT,                    -- 最多65535字节
    article MEDIUMTEXT,             -- 最多1.6千万字节
    big_content LONGTEXT            -- 最多4GB字节
);

INSERT INTO t_string_types (country_code, gender, md5_hash, username, email, address, remark) VALUES
    ('CHN', 'F', 'd41d8cd98f00b204e9800998ecf8427e',
     'alice', 'alice@example.com', 'Guangzhou', 'VIP customer');

-- CHAR vs VARCHAR 空间占用演示
CREATE TABLE t_char_varchar (
    id INT PRIMARY KEY,
    fixed CHAR(10),      -- 固定占10字节（不足用空格填充）
    variable VARCHAR(10) -- 只占实际3字节+长度前缀
);
INSERT INTO t_char_varchar VALUES (1, 'AB', 'AB');
SELECT
    LENGTH(fixed) AS char_length,      -- 10（固定长度）
    LENGTH(variable) AS varchar_length -- 3（实际内容）
FROM t_char_varchar;
```

### 📊 CHAR vs VARCHAR 选择原则

```
┌─────────────────────────────────────────────────────────┐
│ 选 CHAR 的场景          │ 选 VARCHAR 的场景             │
├─────────────────────────────────────────────────────────┤
│ 固定长度（国家码、邮编） │ 可变长度（姓名、地址）         │
│ 非常短的值（1-9字节）   │ 较长字符串                    │
│ 更新频繁的字符串        │ 插入后少更新                  │
│ 存储 MD5 哈希值         │ 存储 JSON/文章内容            │
└─────────────────────────────────────────────────────────┘
```

### ❌ 常见错误

- ❌ VARCHAR 不设最大长度 → 可能达到 65535 字节上限
- ❌ 存 JSON 用 TEXT 而不是 JSON 类型 → 无法做 JSON 函数查询

## 3. 日期时间类型

### 🏠 生活类比

> **DATETIME** = 写在纸上的时间（2024年1月15日 12:30:45），和时区无关
> **TIMESTAMP** = 手机的自动同步时间（会自动根据时区调整）

### 📖 概念讲解

| 类型 | 格式 | 字节 | 时区 | 范围 |
|------|------|------|------|------|
| DATE | YYYY-MM-DD | 3 | ❌ | 1000-9999年 |
| TIME | HH:MM:SS | 3 | ❌ | -838:59:59~838:59:59 |
| YEAR | YYYY | 1 | ❌ | 1901-2155 |
| DATETIME | YYYY-MM-DD HH:MM:SS | 8 | ❌ | 1000-9999年 |
| TIMESTAMP | YYYY-MM-DD HH:MM:SS | 4 | ✅ | 1970-2038年 |

### 💻 SQL 代码示例

```sql
USE data_types_demo;

CREATE TABLE t_datetime_types (
    id INT PRIMARY KEY AUTO_INCREMENT,
    birth_date DATE,                        -- 日期
    work_time TIME,                         -- 时间
    duration TIME,                          -- 时间段
    birth_year YEAR,                        -- 年份
    created_datetime DATETIME,              -- 日期时间（无时区）
    updated_timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP  -- 自动维护时间戳
);

INSERT INTO t_datetime_types
(birth_date, work_time, duration, birth_year, created_datetime)
VALUES
    ('1990-05-20', '09:00:00', '08:30:00', '1990', NOW());

SELECT * FROM t_datetime_types;

-- TIMESTAMP vs DATETIME 时区演示
CREATE TABLE t_timezone_test (
    id INT PRIMARY KEY,
    dt DATETIME,
    ts TIMESTAMP
);
INSERT INTO t_timezone_test VALUES (1, '2024-01-15 12:00:00', '2024-01-15 12:00:00');

-- 查看当前时区
SHOW VARIABLES LIKE 'time_zone';
```

### ❌ 常见错误

- ❌ 存储未来时间（2038年后）用 TIMESTAMP → 超范围，应改用 DATETIME
- ❌ 不区分时区的数据用 TIMESTAMP → DATETIME 更合适

## 4. DECIMAL vs FLOAT 精度问题

### 🏠 生活类比

> **DECIMAL** = 用手算：每一位数字都精确保留（0.1 + 0.2 = 0.3）
> **FLOAT** = 用近似工具算：可能有误差（0.1 + 0.2 ≈ 0.30000000000000004）

### 📖 概念讲解

**IEEE 754 浮点数标准**：某些十进制小数无法用二进制精确表示，只能近似。

`0.1 + 0.2` 在 FLOAT 中 ≈ `0.30000000000000004`

### 💻 SQL 代码示例

```sql
USE data_types_demo;

-- FLOAT 精度丢失演示
CREATE TABLE t_float_precision (
    id INT PRIMARY KEY,
    amount_float FLOAT,
    amount_decimal DECIMAL(10,2)
);
INSERT INTO t_float_precision VALUES
    (1, 0.07, 0.07), (2, 0.05, 0.05),
    (3, 0.03, 0.03), (4, 0.01, 0.01);

SELECT
    id,
    amount_float,
    SUM(amount_float) AS float_sum,
    amount_decimal,
    SUM(amount_decimal) AS decimal_sum
FROM t_float_precision
GROUP BY id;
-- FLOAT 可能显示近似值，DECIMAL 显示精确值

-- 银行转账场景：连续多次扣款
CREATE TABLE t_account_money (
    id INT PRIMARY KEY,
    balance_float FLOAT,
    balance_decimal DECIMAL(15,2)
);
INSERT INTO t_account_money VALUES (1, 1000.00, 1000.00);

-- 连续10次扣 0.1 元，应该剩 999 元
UPDATE t_account_money SET balance_float = balance_float - 0.1;
UPDATE t_account_money SET balance_decimal = balance_decimal - 0.1;

SELECT balance_float, balance_decimal FROM t_account_money;
-- FLOAT 可能显示 999.0000000000001 或 998.9999999999999
-- DECIMAL 精确显示 999.00
```

### ❌ 常见错误

- ❌ 金融计算用 FLOAT/DOUBLE → 精度丢失，金额必须用 DECIMAL

## 5. ENUM 和 SET

### 🏠 生活类比

> **ENUM** = 单选下拉框（性别：男/女/未知）
> **SET** = 多选标签（技能：Python, SQL, JavaScript）

### 📖 概念讲解

| 类型 | 说明 | 最大成员数 | 存储 |
|------|------|-----------|------|
| ENUM | 单选枚举 | 65535 | 1-2 字节整数 |
| SET | 多选集合 | 64 | 位图（8 成员=1 字节） |

### 💻 SQL 代码示例

```sql
USE data_types_demo;

CREATE TABLE t_enum_set_demo (
    id INT PRIMARY KEY AUTO_INCREMENT,

    -- ENUM：单选
    gender ENUM('Male', 'Female', 'Unknown'),
    order_status ENUM('pending', 'paid', 'shipped', 'delivered', 'cancelled'),

    -- SET：多选
    roles SET('admin', 'editor', 'author', 'viewer'),
    permissions SET('read', 'write', 'delete', 'admin')
);

INSERT INTO t_enum_set_demo (gender, order_status, roles, permissions) VALUES
    ('Female', 'paid', 'admin,editor', 'read,write'),
    ('Male', 'shipped', 'author', 'read'),
    ('Unknown', 'pending', 'viewer', 'read');

SELECT * FROM t_enum_set_demo;

-- SET 查询
SELECT * FROM t_enum_set_demo WHERE FIND_IN_SET('admin', roles) > 0;
SELECT * FROM t_enum_set_demo WHERE FIND_IN_SET('write', permissions) > 0;
```

### ❌ 常见错误

- ❌ ENUM/SET 选项会频繁变化 → 改用 VARCHAR 更灵活（ALTER TABLE 成本高）

## 6. BLOB 和 TEXT

### 📖 概念讲解

| 类型 | 最大存储 | 特点 |
|------|---------|------|
| TINYBLOB/BLOB/MEDIUMBLOB/LONGBLOB | 255B/64KB/16MB/4GB | 原始字节流，无字符集 |
| TINYTEXT/TEXT/MEDIUMTEXT/LONGTEXT | 255B/64KB/16MB/4GB | 字符数据，有字符集 |

**BLOB** = 图片、音频、视频、压缩数据、加密数据
**TEXT** = 文章内容、日志、JSON/XML 大文档

### 💻 SQL 代码示例

```sql
USE data_types_demo;

CREATE TABLE t_blob_text_demo (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(100),
    content TEXT,        -- 文章正文
    attachment BLOB     -- 二进制附件
);

INSERT INTO t_blob_text_demo (title, content) VALUES
    ('Article 1', '这是一篇很长的文章内容...'),
    ('Article 2', '另一篇文章内容...');

-- TEXT 的字符集影响排序
CREATE TABLE t_text_charset (
    id INT PRIMARY KEY,
    content TEXT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci
);
INSERT INTO t_text_charset VALUES (1, '中国');
SELECT * FROM t_text_charset;
```

### ❌ 常见错误

- ❌ TEXT 列在 InnoDB 中放最后 → InnoDB 把大列存溢出页，放最后可优化
- ❌ BLOB/TEXT 不设字符集 → 可能导致乱码

## 7. UNSIGNED 整数

### 📖 概念讲解

**UNSIGNED** = 无符号数，只能表示非负整数，取值范围**翻倍**。

| 类型 | 有符号 | 无符号 |
|------|--------|--------|
| TINYINT | -128~127 | 0~255 |
| INT | -21亿~21亿 | 0~42亿 |

### 💻 SQL 代码示例

```sql
USE data_types_demo;

CREATE TABLE t_unsigned_demo (
    id INT UNSIGNED PRIMARY KEY,      -- 用户ID不会是负数
    age TINYINT UNSIGNED,             -- 年龄 0-255 足够
    view_count INT UNSIGNED DEFAULT 0  -- 浏览次数
);

INSERT INTO t_unsigned_demo VALUES
    (1, 25, 100),
    (4294967295, 255, 999999999);  -- INT UNSIGNED 最大值

SELECT * FROM t_unsigned_demo;
```

### ❌ 常见错误

- ❌ 给已插入负数的列加 UNSIGNED → 报错，数据类型变更要谨慎

## 8. DEFAULT 默认值

### 📖 概念讲解

| 字段情况 | 默认值 |
|---------|--------|
| 有 DEFAULT 约束 | 使用默认值 |
| 允许 NULL 但无 DEFAULT | NULL |
| NOT NULL 且无 DEFAULT | 报错（必须显式提供值） |

### 💻 SQL 代码示例

```sql
USE data_types_demo;

CREATE TABLE t_default_demo (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,

    -- 数字默认值
    age INT DEFAULT 0,
    score DECIMAL(5,1) DEFAULT 0.0,

    -- 字符串默认值
    phone VARCHAR(20) DEFAULT '' NOT NULL,
    bio VARCHAR(200) DEFAULT NULL,
    status VARCHAR(20) DEFAULT 'active',

    -- 日期时间默认值
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    -- 表达式默认值（MySQL 5.7+）
    total_amount DECIMAL(10,2) DEFAULT (100.00 * 2)
);

INSERT INTO t_default_demo (name) VALUES ('Alice');
INSERT INTO t_default_demo (name, age, phone, status) VALUES ('Bob', 30, '13800138000', 'inactive');

SELECT * FROM t_default_demo;
-- Alice: age=0, score=0.0, phone='', bio=NULL, status='active'
```

### ❌ 常见错误

- ❌ NOT NULL 列不加 DEFAULT → 每次插入都要显式提供值

## 本章小结

```
┌─────────────────────────────────────────────────────────┐
│                    数据类型选择指南                      │
├─────────────────────────────────────────────────────────┤
│ 整数：TINYINT（年龄）→ SMALLINT → INT（一般ID）→ BIGINT │
│ 金额：DECIMAL(10,2)（精确存储）                         │
│ 科学计算：FLOAT/DOUBLE（允许近似值）                    │
│                                                          │
│ 字符串：CHAR(固定) vs VARCHAR(可变)                      │
│   固定短串（邮编/MD5）→ CHAR                             │
│   可变中长串（姓名/地址）→ VARCHAR                      │
│   大文本 → TEXT/MEDIUMTEXT/LONGTEXT                    │
│                                                          │
│ 日期：DATE（日期）/ DATETIME（无时区）/ TIMESTAMP（有时区）│
│   未来时间>2038年 → DATETIME                           │
│   跨时区时间 → TIMESTAMP                                 │
│   精确金额 → DECIMAL（绝对不能是 FLOAT）                │
│                                                          │
│ ENUM = 单选 / SET = 多选 / BLOB = 纯二进制              │
└─────────────────────────────────────────────────────────┘
```

## 课后练习

**练习 1：DECIMAL vs FLOAT 精度对比**
创建包含 DECIMAL(10,2)、FLOAT、DOUBLE 类型的表，分别插入值 0.1，连续做 10 次加 0.1 运算，观察精度差异。

**练习 2：DATETIME vs TIMESTAMP 存储差异**
用 DATETIME 和 TIMESTAMP 各创建一张表，插入相同时间值，用 UNION ALL 观察两种类型的存储差异。

**练习 3：ENUM 排序验证**
创建 ENUM 类型存储订单状态，插入后用 ORDER BY 排序，验证排序是否按定义顺序而非字母顺序。

**练习 4：CHAR vs VARCHAR 长度测量**
将 VARCHAR(20) 和 CHAR(20) 分别插入 'abc' 和 '  abc  '（带空格），用 LENGTH() 和 CHAR_LENGTH() 测量，解释差异。

**练习 5：设计登录日志表**
包含：用户 ID、登录时间（精确到毫秒）、登录 IP、登录设备类型。合理选择数据类型并说明理由。
