# 模块十一：多表查询

## 学习目标

- 理解表与表之间的三种关系（一对一、一对多、多对多）
- 掌握 INNER JOIN / LEFT JOIN / RIGHT JOIN 的用法
- 理解 ON vs WHERE 在 JOIN 中的区别
- 掌握子查询在多表查询中的应用
- 掌握 IN 和 EXISTS 的区别和选择
- 掌握自连接的原理和应用场景

---

## 1. 表与表之间的关系

### 🏠 生活类比

> **一对一** = 一个人对应一个身份证号
> **一对多** = 一个班级对应多个学生
> **多对多** = 一个学生对应多门课程，一门课程对应多个学生

### 📖 概念讲解

| 关系类型 | 实现方式 | 典型场景 |
|---------|---------|---------|
| 一对一（1:1） | 主键关联 或 外键唯一约束 | 用户表-用户详情表 |
| 一对多（1:N） | 外键不加唯一约束 | 部门-员工 |
| 多对多（M:N） | 中间表（两个外键） | 学生-课程（通过选课表） |

### 💻 SQL 代码示例

```sql
CREATE DATABASE multi_table_demo;
USE multi_table_demo;

-- 一对一关系：用户表和用户详情表（主键关联）
CREATE TABLE t_user (
    user_id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50)
);
CREATE TABLE t_user_profile (
    user_id INT PRIMARY KEY,                  -- 主键，同时是外键
    real_name VARCHAR(50),
    phone VARCHAR(20),
    FOREIGN KEY (user_id) REFERENCES t_user(user_id)
);
INSERT INTO t_user VALUES (1, 'alice'), (2, 'bob');
INSERT INTO t_user_profile VALUES
    (1, 'Alice Smith', '13812345678'),
    (2, 'Bob Wang', '13998765432');

-- 一对多关系：部门和员工
CREATE TABLE t_dept (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50)
);
CREATE TABLE t_emp (
    emp_id INT PRIMARY KEY AUTO_INCREMENT,
    emp_name VARCHAR(50),
    dept_id INT,                             -- 外键，不唯一
    FOREIGN KEY (dept_id) REFERENCES t_dept(dept_id)
);
INSERT INTO t_dept VALUES (1, 'IT'), (2, 'HR');
INSERT INTO t_emp VALUES
    (1, 'Alice', 1), (2, 'Bob', 1), (3, 'Carol', 2);

-- 多对多关系：学生和课程（通过选课表关联）
CREATE TABLE t_student (
    student_id INT PRIMARY KEY,
    student_name VARCHAR(50)
);
CREATE TABLE t_course (
    course_id INT PRIMARY KEY,
    course_name VARCHAR(50)
);
CREATE TABLE t_enrollment (
    student_id INT,
    course_id INT,
    grade DECIMAL(4,1),
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (student_id) REFERENCES t_student(student_id),
    FOREIGN KEY (course_id) REFERENCES t_course(course_id)
);
INSERT INTO t_student VALUES
    (1, 'Alice'), (2, 'Bob'), (3, 'Carol');
INSERT INTO t_course VALUES
    (101, 'Math'), (102, 'English'), (103, 'Physics');
INSERT INTO t_enrollment VALUES
    (1, 101, 85), (1, 102, 90),
    (2, 101, 78), (2, 103, 92),
    (3, 102, 88);
```

### 📊 ASCII 关系图

```
一对一（1:1）：
t_user ───────── t_user_profile
   PK ◄────────► PK+FK

一对多（1:N）：
t_dept ───────── t_emp
  PK ◄──────── FK (不唯一)

多对多（M:N）：
t_student ── t_enrollment ── t_course
    PK ◄─── FK1       FK2 ──► PK
```

### ❌ 常见错误

- ❌ 多对多关系不用中间表 → 直接关联会导致数据冗余和更新异常

## 2. INNER JOIN

### 🏠 生活类比

> **INNER JOIN** = 两份名单取交集：只在两份名单里都出现的人

### 📖 概念讲解

`INNER JOIN` = 返回两表中连接条件匹配的行。只保留两表都满足条件的记录。

### 💻 SQL 代码示例

```sql
USE multi_table_demo;

-- 基础 INNER JOIN
SELECT e.emp_name, d.dept_name
FROM t_emp e
INNER JOIN t_dept d ON e.dept_id = d.dept_id;

-- 多表 INNER JOIN
SELECT e.emp_name, d.dept_name, p.real_name, p.phone
FROM t_emp e
INNER JOIN t_dept d ON e.dept_id = d.dept_id
INNER JOIN t_user u ON e.emp_name = u.username
INNER JOIN t_user_profile p ON u.user_id = p.user_id;

-- INNER JOIN 等价于隐式连接（逗号+WHERE）
SELECT e.emp_name, d.dept_name
FROM t_emp e, t_dept d
WHERE e.dept_id = d.dept_id;

-- 场景：查询选课学生信息和课程信息
SELECT s.student_name, c.course_name, e.grade
FROM t_student s
INNER JOIN t_enrollment e ON s.student_id = e.student_id
INNER JOIN t_course c ON e.course_id = c.course_id;
```

### ❌ 常见错误

- ❌ 忘记写 JOIN 条件 → 笛卡尔积（数据量爆炸式增长）

## 3. LEFT JOIN 和 RIGHT JOIN

### 🏠 生活类比

> **LEFT JOIN** = 以左边的名单为准，右边没有的显示为空
> **RIGHT JOIN** = 以右边的名单为准，左边没有的显示为空

### 📖 概念讲解

| 类型 | 说明 | 常用场景 |
|------|------|---------|
| LEFT JOIN | 返回左表所有行，右表不匹配显示 NULL | 最常用 |
| RIGHT JOIN | 返回右表所有行，左表不匹配显示 NULL | 较少用 |

### 💻 SQL 代码示例

```sql
USE multi_table_demo;

-- LEFT JOIN：所有部门，包括没有员工的部门
INSERT INTO t_dept VALUES (3, 'Marketing');  -- 插入没有员工的部门

SELECT d.dept_name, e.emp_name
FROM t_dept d
LEFT JOIN t_emp e ON d.dept_id = e.dept_id;
-- IT部门有Alice和Bob，HR部门有Carol，Marketing部门显示NULL

-- 场景：统计每个部门的员工数量（含0个员工的部门）
SELECT d.dept_name, COUNT(e.emp_id) AS emp_count
FROM t_dept d
LEFT JOIN t_emp e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name;
-- IT=2, HR=1, Marketing=0

-- LEFT JOIN + WHERE：查找没有员工的部门
SELECT d.dept_name
FROM t_dept d
LEFT JOIN t_emp e ON d.dept_id = e.dept_id
WHERE e.emp_id IS NULL;
```

### 📊 ASCII 图：LEFT JOIN

```
t_dept (左)          LEFT JOIN t_emp (右)

┌─────────┬──────────────────────────┐
│ dept_name│ emp_name                 │
├─────────┼──────────────────────────┤
│ IT      │ Alice                    │
│ IT      │ Bob                      │
│ HR      │ Carol                    │
│ Marketing│ NULL ← 右表无匹配，显示NULL│
└─────────┴──────────────────────────┘
```

### ❌ 常见错误

- ❌ 混淆 LEFT JOIN 和 INNER JOIN → 想保留左表所有行时用了 INNER JOIN

## 4. 多表连接注意事项

### 💻 SQL 代码示例

```sql
USE multi_table_demo;

-- 多表 JOIN 示例
SELECT
    s.student_name,
    c.course_name,
    e.grade,
    d.dept_name
FROM t_student s
INNER JOIN t_enrollment e ON s.student_id = e.student_id
INNER JOIN t_course c ON e.course_id = c.course_id
INNER JOIN t_emp emp ON s.student_name = emp.emp_name
INNER JOIN t_dept d ON emp.dept_id = d.dept_id
WHERE e.grade >= 80
ORDER BY e.grade DESC;

-- ON vs WHERE 的区别
INSERT INTO t_dept VALUES (4, 'Finance');
INSERT INTO t_emp VALUES (4, 'David', NULL);  -- David没有部门

-- ON：连接时不管 dept_id 是否为 NULL
SELECT e.emp_name, d.dept_name
FROM t_emp e
LEFT JOIN t_dept d ON e.dept_id = d.dept_id;
-- David 显示 NULL

-- WHERE：过滤掉不匹配的行
SELECT e.emp_name, d.dept_name
FROM t_emp e
LEFT JOIN t_dept d ON e.dept_id = d.dept_id
WHERE d.dept_name IS NOT NULL;
-- David 被过滤掉了（因为 dept_id=NULL，ON 不会匹配）
```

### ❌ 常见错误

- ❌ LEFT JOIN 后在 WHERE 中加右表条件 → 会把 LEFT JOIN 变成 INNER JOIN

## 5. 子查询在多表查询中的应用

### 💻 SQL 代码示例

```sql
USE multi_table_demo;

-- 场景1：查找选了Math课程的学生
SELECT s.student_name
FROM t_student s
WHERE s.student_id IN (
    SELECT course_id FROM t_course WHERE course_name = 'Math'  -- 应该是 student_id
);
-- 修正：子查询查的是 enrollment 中的 student_id

SELECT s.student_name
FROM t_student s
WHERE s.student_id IN (
    SELECT student_id FROM t_enrollment
    WHERE course_id = (SELECT course_id FROM t_course WHERE course_name = 'Math')
);

-- 场景2：查找分数高于平均分的学生
SELECT s.student_name, e.grade
FROM t_student s
INNER JOIN t_enrollment e ON s.student_id = e.student_id
WHERE e.grade > (SELECT AVG(grade) FROM t_enrollment);

-- 场景3：FROM 子查询（派生表）
SELECT avg_stats.grade_level, COUNT(*) AS student_count
FROM (
    SELECT
        s.student_name,
        AVG(e.grade) AS avg_grade,
        CASE
            WHEN AVG(e.grade) >= 90 THEN 'A'
            WHEN AVG(e.grade) >= 80 THEN 'B'
            ELSE 'C'
        END AS grade_level
    FROM t_student s
    INNER JOIN t_enrollment e ON s.student_id = e.student_id
    GROUP BY s.student_id, s.student_name
) AS avg_stats
GROUP BY avg_stats.grade_level;
```

## 6. IN 和 EXISTS 的区别

### 📖 概念讲解

| 操作符 | 执行方式 | 性能特点 |
|--------|---------|---------|
| IN | 先执行子查询得到列表，再匹配 | 列表大时性能下降 |
| EXISTS | 对外层每一行，检查内层是否有匹配 | 外层大、子层小时更高效 |

### 💻 SQL 代码示例

```sql
USE multi_table_demo;

-- IN：查找选了Math或English课程的学生
SELECT s.student_name
FROM t_student s
WHERE s.student_id IN (
    SELECT e.student_id FROM t_enrollment e
    INNER JOIN t_course c ON e.course_id = c.course_id
    WHERE c.course_name IN ('Math', 'English')
);

-- EXISTS：查找选了课程的学生
SELECT s.student_name
FROM t_student s
WHERE EXISTS (
    SELECT 1 FROM t_enrollment e WHERE e.student_id = s.student_id
);

-- NOT IN vs NOT EXISTS
-- 查找没选任何课的学生
SELECT s.student_name
FROM t_student s
WHERE s.student_id NOT IN (
    SELECT DISTINCT student_id FROM t_enrollment
);

SELECT s.student_name
FROM t_student s
WHERE NOT EXISTS (
    SELECT 1 FROM t_enrollment e WHERE e.student_id = s.student_id
);
```

### ❌ 常见错误

- ❌ IN 的子查询返回 NULL → `WHERE col IN (NULL)` 永远不会匹配

## 7. 自连接

### 🏠 生活类比

> **自连接** = 镜子对着镜子：自己和自己连，照出无限反射

### 📖 概念讲解

自连接 = 一个表与自己进行连接，需要给表起两个不同的别名。

典型场景：员工表（含上级领导 ID）、分类表（含父类 ID）、好友关系

### 💻 SQL 代码示例

```sql
USE multi_table_demo;

-- 员工表（含上级领导 ID）
CREATE TABLE t_employee (
    emp_id INT PRIMARY KEY,
    emp_name VARCHAR(50),
    manager_id INT,              -- 上级领导 ID
    salary DECIMAL(10,2)
);
INSERT INTO t_employee VALUES
    (1, 'CEO Zhang', NULL, 100000.00),
    (2, 'VP Wang', 1, 80000.00),
    (3, 'VP Li', 1, 80000.00),
    (4, 'Manager Chen', 2, 50000.00),
    (5, 'Manager Wu', 3, 50000.00),
    (6, 'Staff Sun', 4, 30000.00),
    (7, 'Staff Zhao', 5, 30000.00);

-- 场景1：查询每个员工及其上级名字
SELECT
    e.emp_name AS employee,
    m.emp_name AS manager
FROM t_employee e
LEFT JOIN t_employee m ON e.manager_id = m.emp_id;

-- 场景2：查询每个经理的下属数量
SELECT
    m.emp_name AS manager,
    COUNT(e.emp_id) AS subordinate_count
FROM t_employee e
INNER JOIN t_employee m ON e.manager_id = m.emp_id
GROUP BY m.emp_id, m.emp_name;

-- 场景3：查询所有层级关系
SELECT
    L1.emp_name AS level1,    -- CEO
    L2.emp_name AS level2,    -- VP
    L3.emp_name AS level3,    -- Manager
    L4.emp_name AS level4     -- Staff
FROM t_employee L1
LEFT JOIN t_employee L2 ON L2.manager_id = L1.emp_id
LEFT JOIN t_employee L3 ON L3.manager_id = L2.emp_id
LEFT JOIN t_employee L4 ON L4.manager_id = L3.emp_id
WHERE L1.manager_id IS NULL;  -- 从CEO开始
```

### 📊 ASCII 自连接示意

```
t_employee（自连接）：

CEO Zhang (emp_id=1, manager_id=NULL)
│
├── VP Wang (emp_id=2, manager_id=1)
│   └── Manager Chen (emp_id=4, manager_id=2)
│       └── Staff Sun (emp_id=6, manager_id=4)
└── VP Li (emp_id=3, manager_id=1)
    └── Manager Wu (emp_id=5, manager_id=3)
        └── Staff Zhao (emp_id=7, manager_id=5)
```

### ❌ 常见错误

- ❌ 自连接忘加条件 → 笛卡尔积

## 本章小结

```
┌─────────────────────────────────────────────────────────┐
│                    多表查询核心                           │
├─────────────────────────────────────────────────────────┤
│ 表关系：一对一（主键关联）/ 一对多（外键）/ 多对多（中间表）│
│                                                          │
│ INNER JOIN = 两表都匹配才返回                           │
│ LEFT JOIN = 保留左表所有行，右表无匹配显示NULL           │
│ RIGHT JOIN = 保留右表所有行，左表无匹配显示NULL           │
│                                                          │
│ ON vs WHERE：                                           │
│   ON决定连接，WHERE决定过滤                            │
│   LEFT JOIN + WHERE右表条件 = 变成INNER JOIN           │
│                                                          │
│ IN vs EXISTS：                                          │
│   IN：列表匹配，适合子查询结果小                       │
│   EXISTS：存在性检查，适合外层结果小                    │
│                                                          │
│ 自连接：表与自身连接，需要两个别名                     │
│   典型场景：组织架构、分类树、好友关系               │
└─────────────────────────────────────────────────────────┘
```
